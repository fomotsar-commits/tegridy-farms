================================================================================
TEGRIDY FARMS — FULL CODEBASE OVERVIEW
================================================================================

Tech Stack: React 19 + TypeScript + Vite 8 + Tailwind CSS 4 + wagmi/viem + RainbowKit

================================================================================
SPLASH SCREEN / LOADER (components/loader/)
================================================================================

Canvas-based animated intro sequence. Phases run sequentially:

1. VOID (1200ms) — Purple mist, golden horizontal lines
2. ART (5000ms × image count) — Gallery of art with zoom/fade/glitch between pieces
3. SHATTER (1000ms) — Particles explode outward from last art piece
4. VORTEX (2000ms) — Particles spiral inward following mouse/touch
5. TEXT-FORM (2000ms) — Particles form "TEGRIDY" and "FARMS" text via spring physics
6. HOLD (until click) — Breathing particles, mouse trail, "CLICK TO ENTER"
7. EXIT-CRACK (500ms) — Branching cracks grow from click point
8. EXIT (variable) — Ragdoll shard physics, fade to main app
9. SKIP (400ms) — ESC key dissolve animation

Key Implementation Details:
  - 2000 particles on desktop, 1000 on mobile
  - Text pixel targets generated by rendering text to offscreen canvas, sampling filled pixels
  - Targets shuffled before assignment so particles distribute evenly across text
  - Mobile: no WebGL post-processing, no particle morphing, no backdrop-filter
  - Audio: Web Audio API procedural SFX (crack, shatter) + ambient MP3 loop
  - Skipped on repeat visits via sessionStorage('tf_loaded')

================================================================================
GLITCH TRANSITION (components/GlitchTransition.tsx)
================================================================================

Dual rendering paths:

MOBILE (single <canvas>):
  - Art slices with chromatic aberration (RGB channel offsets)
  - Block displacement, noise grain, scanlines
  - Subliminal text flash
  - VHS tracking band
  - White flash at peak
  - Optimized for Safari iOS

DESKTOP (DOM-based):
  - 8-16 animated art slices with per-slice keyframes
  - Canvas noise overlay (32ms refresh)
  - Horizontal sweep lines
  - Subliminal text with scale/rotation animation
  - Screen flash, scanline overlay, afterimage blur

Config varies by route:
  Homepage transitions: heavy intensity, 16 slices, 2800ms (desktop) / 8 slices, 2000ms (mobile)
  Adjacent pages: light, 8 slices, 1800ms / 5 slices, 1400ms
  Other: medium, 12 slices, 2200ms / 6 slices, 1600ms

================================================================================
PRICE FEED CHAIN
================================================================================

1. Primary: GeckoTerminal API (real-time)
2. Secondary: Uniswap V2 LP reserves (on-chain getReserves)
3. Fallback: localStorage cached price (10-min TTL)
4. ETH/USD: Chainlink oracle (1-hour staleness limit)

================================================================================
KNOWN MOBILE/SAFARI CONSIDERATIONS
================================================================================

- mix-blend-mode broken inside position:fixed on Safari → use opacity/filter alternatives
- CSS custom properties (var()) ignored inside @keyframes on Safari → use JS rAF animations
- Canvas shadowBlur renders much brighter on Safari → removed ghost text on mobile
- Safari aggressively caches index.html → cache-control headers in vercel.json
- Particle count reduced on mobile (1000 vs 2000) for performance
- WebGL post-processing disabled on mobile
- Touch events used alongside mouse events for interactive features
