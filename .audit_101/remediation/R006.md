# R006 — react-hooks/rules-of-hooks violations (P0-HIGH)

**Source finding:** agent 096 — `frontend/npm run lint` reports 25 `react-hooks/rules-of-hooks` errors (`useReadContract`, `useMemo`, `useState`, `useEffect`, `useCallback`, `useReadContracts`, custom `useCountdown`) called conditionally. React 19 rules: hooks must be called unconditionally at the top of every render.

**Patterns applied (per the React docs):**
- **Pattern D** (split component): wrap an early-return guard around an inner component so hooks always run in the inner component.
- **Pattern B** (hoist hooks above early returns): move trailing hooks to before the conditional return.
- **Pattern C** (hoist custom hook + reuse value): call `useCountdown` once at the top, reference its return value inside conditional JSX.
- **eslint-disable line comment** for a single false positive in a Playwright fixture (the rule mis-flags the Playwright `use` parameter as a React hook).

## Files touched

| File | Pattern | Change |
|---|---|---|
| `frontend/src/components/GaugeVoting.tsx` | D + C | Split `GaugeVoting` into a thin wrapper that handles the not-deployed early return, and a new `GaugeVotingInner` that holds every hook (22 sites: `useReadContract` x10, `useReadContracts`, `useMemo` x3, `useState` x2, `useEffect` x2, `useCallback` x2, custom `useCountdown` x2). The `useCountdown(revealOpensAt)` previously called inline inside conditional JSX is now hoisted to the component top as `revealCountdown`; the JSX references the hoisted value. |
| `frontend/src/components/nftfinance/AMMSection.tsx` | B | In `PoolCard`, hoisted three trailing hooks (`useState` x2 for `adminExpanded`/`historyExpanded`, `useCallback` for `refetchAll`) above the `if (!poolInfo) return …loading skeleton…` early return. Removed the duplicate declarations from below the return. |
| `frontend/e2e/fixtures/wallet.ts` | eslint-disable | The Playwright `test.extend` API requires a callback parameter literally named `use`; the eslint react-hooks rule pattern-matches that identifier and mis-flags it. Added a single-line `eslint-disable-next-line react-hooks/rules-of-hooks` with an explanatory comment. Not a React hook. |

## Before / after sketch

### `GaugeVoting.tsx` (Pattern D)

**Before:**
```tsx
export function GaugeVoting() {
  const { address, isConnected } = useAccount();
  const chainId = useChainId();
  const [weights, setWeights] = useState(...);
  const [mode, setMode] = useState(...);
  const { writeContract, data: txHash, isPending: isSigning } = useWriteContract();
  const { isLoading: isConfirming, isSuccess } = useWaitForTransactionReceipt(...);

  if (!isDeployed(GAUGE_CONTROLLER_ADDRESS)) {
    return <NotDeployedPlaceholder />;     // ← early return
  }

  // ↓ all of these were AFTER the early return → conditionally called
  const { data: epochData } = useReadContract(...);
  const { data: budgetData } = useReadContract(...);
  // ...
  const countdown = useCountdown(nextEpochTimestamp);
  // ...
  // and inside JSX:
  <p>Reveal opens in {useCountdown(revealOpensAt)}</p>   // ← conditional hook call
}
```

**After:**
```tsx
export function GaugeVoting() {
  if (!isDeployed(GAUGE_CONTROLLER_ADDRESS)) {
    return <NotDeployedPlaceholder />;
  }
  return <GaugeVotingInner />;
}

function GaugeVotingInner() {
  const { address, isConnected } = useAccount();
  const chainId = useChainId();
  // ... all hooks unconditionally at the top, in the same order ...
  const countdown = useCountdown(nextEpochTimestamp);
  const revealCountdown = useCountdown(revealOpensAt);   // ← hoisted, called every render
  // ... non-hook conditional logic and `if (!isConnected) return` come AFTER all hooks ...
  return <p>Reveal opens in {revealCountdown}</p>;       // ← reference, not call
}
```

### `AMMSection.tsx` (Pattern B)

**Before:**
```tsx
function PoolCard({ poolAddress, ... }) {
  const { address } = useAccount();
  const [expanded, setExpanded] = useState(false);
  // ... other hooks ...
  const { data: poolInfo, refetch: refetchPoolInfo } = useReadContract(...);

    return <Skeleton />;                          // ← early return
  }

  // ↓ AFTER the early return → conditionally called
  const [adminExpanded, setAdminExpanded] = useState(false);
  const [historyExpanded, setHistoryExpanded] = useState(false);
  const refetchAll = useCallback(() => { ... }, [...]);
  return <m.div>...</m.div>;

```

**After:**
```tsx
function PoolCard({ poolAddress, ... }) {
  const { address } = useAccount();
  const [expanded, setExpanded] = useState(false);
  // ... other hooks ...
  // Hoisted above early-return so hooks always run (R006)
  const [adminExpanded, setAdminExpanded] = useState(false);
  const [historyExpanded, setHistoryExpanded] = useState(false);
  const { data: poolInfo, refetch: refetchPoolInfo } = useReadContract(...);
  // ...
  const refetchAll = useCallback(() => { ... }, [...]);

  if (!poolInfo) {
    return <Skeleton />;
  }
  return <m.div>...</m.div>;
}
```

**After:**

```ts
    // ... build mock ...
    // Playwright fixture API: `use` is the fixture-callback parameter, not a React hook.

    await use(mock);
  },

```

## Behavior preserved

No user-visible behavior change.

- `GaugeVoting` still renders the not-deployed placeholder when the gauge controller is undeployed; otherwise renders the full UI. Hooks now always run when `GaugeVotingInner` mounts (i.e. after the not-deployed branch); when the controller IS deployed, the inner component mounts exactly as before. No new fetches happen in the undeployed branch.
- `PoolCard` still shows the loading skeleton until `poolInfo` arrives, then renders the full card. The hoisted `useState`/`useCallback` calls are pure (no fetching) — they always ran on the second render anyway, just in a misordered way that React flagged but didn't crash. Hoisting makes the order stable.
- `e2e/fixtures/wallet.ts` is a Playwright test fixture, not a React component — disabling the rule on the one line that triggers the false positive does not change runtime behavior.
