# R039 — Modal focus trap, OnboardingModal hardening, z-index unification

## Source findings (agent 069, HIGH)

- **H-1** `OnboardingModal` auto-opens on first visit, lacked focus trap,
  scroll lock, and focus restoration. Worse: backdrop tap dismissed the
  modal AND set `tegridy-onboarding-seen=1` in localStorage, so on iPhone
  Safari a scroll-fling on the dialog body would silently mark a user as
  "onboarded" before they had read anything.
- **H-2** Base `Modal.tsx` auto-focused the dialog root and locked body
  scroll but installed no Tab / Shift+Tab cycle. Every consumer
  (`TokenSelectModal`, `TraitEditor`, etc.) inherited the regression —
  Tab walked straight through to background page controls. Only
  `ArtLightbox` and the `TopNav` mobile drawer hand-rolled their own
  trap; the shared primitive did not.
- **H-3 / M-1** `BottomNav` `z-50` collided with `ArtLightbox` `z-50`
  (same stacking context, latter rendered later → won, but fragile).
  Also, `TopNav` primary nav was previously `hidden md:flex` so iPad
  portrait (744 px) and small tablets dropped to the cramped
  hamburger + BottomNav combo. R038 (preceding batch) had already fixed
  the breakpoint move from `md:` → `sm:` for both `TopNav` and
  `BottomNav`; R039 finishes the surface by replacing every ad-hoc
  `z-50` / `z-[100]` with a semantic layering scale.

## Battle-tested patterns adopted

- Focus trap implementation mirrors **focus-trap-react** /
  **Headless UI** / **Radix Dialog** — same focusable selector list,
  same Tab/Shift+Tab cycle logic, same `previouslyFocused` save/restore
  pattern. No new dependency added (we already pull in framer-motion
  for animations and would have to wrap the panel in a third-party
  trap component, which collides with `m.div`'s ref handling).
- Z-index scale uses Tailwind v4's `@utility` directive so each
  semantic name (`z-modal-content`, `z-top-nav`, etc.) compiles to a
  real utility class. Variables stay in `@theme` so inline styles can
  reference them too (`style={{ zIndex: 'var(--z-modal-content)' }}`).

## Changes applied

### `frontend/src/index.css` — z-index layering scale (`@theme` + `@utility`)

Added 10 named z-index variables under `@theme`, then 9 corresponding
`@utility` blocks. Tailwind v4 only auto-generates utilities from a
fixed namespace list (color, spacing, font, breakpoint, …) — `--z-*`
is **not** in that list, so explicit `@utility` definitions are
required. Build output verified: `.z-modal-content{z-index:var(--z-modal-content)}`
etc. all land in `dist/assets/index-*.css`.

| Token              | Value | Use                                                    |
| ------------------ | ----- | ------------------------------------------------------ |
| `z-base`           | 0     | normal flow                                            |
| `z-bottom-nav`     | 30    | mobile tab bar — below modals so a lightbox covers it |
| `z-top-nav`        | 40    | fixed header                                           |
| `z-drawer-backdrop`| 45    | mobile nav drawer scrim                                |
| `z-drawer`         | 46    | mobile nav drawer panel                                |
| `z-banner`         | 49    | wrong-network banner under TopNav                      |
| `z-modal-backdrop` | 50    | dialog scrim (covers navs)                             |
| `z-modal-content`  | 51    | dialog panel                                           |
| `z-toast`          | 60    | Sonner toasts above everything                         |
| `z-max`            | 9999  | last-resort (loaders / error boundaries)               |

### `frontend/src/components/ui/Modal.tsx` — full focus trap + dismiss-opt-out

- New `FOCUSABLE_SELECTOR` constant (canonical list).
- New `getFocusableDescendants()` helper that filters disabled,
  `aria-hidden`, and zero-bbox elements at runtime so async-mounted
  controls don't leak into the trap before they paint.
- New `previouslyFocusedRef` — `document.activeElement` is captured on
  open, `.focus()` is invoked on close (try/catch so unmounted nodes
  don't throw).
- Keyboard handler now also handles `Tab` / `Shift+Tab` — wraps from
  last → first / first → last. Edge case: when no focusables exist
  inside the dialog, `Tab` is preventDefault-ed and focus is bounced
  back onto the dialog root so it can never escape.
- Auto-focus deferred via `requestAnimationFrame` so AnimatePresence
  has finished mounting the panel before we focus it (focusing a
  not-yet-rendered node is a silent no-op).
- New optional prop `dismissOnBackdrop?: boolean` (default `true`).
  Controls whether the scrim's `onClick` fires `onClose`. **Public API
  is unchanged for existing consumers** — they get the legacy behavior
  by default.
- Replaced ad-hoc `z-[100]` / `z-[101]` with `z-modal-backdrop` /
  `z-modal-content`.

### `frontend/src/components/ui/OnboardingModal.tsx` — refactor onto base Modal

- Now wraps `<Modal …>` from `./Modal` and inherits the shared focus
  trap, focus restore, body scroll lock, Escape close, and themed
  backdrop. Title moved into the Modal header (associated via
  `aria-labelledby`).
- Passes `dismissOnBackdrop={false}` so a fling-tap on the dialog
  card cannot fire backdrop click → silent dismissal. Users can only
  close via:
  1. the explicit × close button
  2. Escape key
  3. completing the flow (Start Farming / Buy TOWELI CTAs)
- Removed the bespoke Escape handler, scroll lock, and inline backdrop
  div (all now lived inside the base Modal). ~30 lines smaller and no
  longer drifts from the canonical primitive.

### `frontend/src/components/ui/ArtLightbox.tsx`

- `z-50` → `z-modal-content` (collision fix vs BottomNav).

### `frontend/src/components/layout/TopNav.tsx`

- Header `z-50` → `z-top-nav`.
- "More" dropdown `z-50` → `z-top-nav` (sits inside the header
  stacking context).
- Mobile drawer backdrop `z-50` → `z-drawer-backdrop`; drawer panel
  `z-50` → `z-drawer`. Drawer panel is now strictly above its scrim
  (46 > 45) — previously they shared `z-50` and rendered order was
  the only thing keeping the panel on top.
- Note: the `hidden md:flex` → `hidden sm:flex` breakpoint move for
  the primary nav was already shipped in R038 (verified in current
  source); H-3's iPad-portrait nav fix is preserved here, no change
  needed.

### `frontend/src/components/layout/BottomNav.tsx`

- `z-50` → `z-bottom-nav` (30). Now strictly below modal scrim, so
  ArtLightbox / Modal opening over it visibly covers the tab bar
  instead of relying on render order.

### `frontend/src/components/layout/AppLayout.tsx`

- Wrong-network banner `z-50` → `z-banner` (49). Sits just under the
  modal scrim so an open dialog hides the banner instead of showing a
  bright red strip through the dialog top edge.

### `frontend/src/components/ui/OnboardingModal.test.tsx`

- Wrapped `<OnboardingModal />` in `<ThemeProvider>` so the new base
  Modal's `useTheme()` hook resolves in tests (production already
  wraps it in `AppLayout`).
- "Close button (x)" — updated `getByLabelText('Close')` →
  `getByLabelText('Close dialog')` (label inherited from base Modal).
- "Clicking backdrop overlay closes modal" → renamed and inverted to
  "Clicking backdrop overlay does NOT close modal (R039)" — asserts
  `localStorage` stays `null` and the welcome step is still rendered
  after a backdrop click. Documents the H-1 regression in the test
  body so future contributors understand why the assertion is
  inverted.

## Notes / non-changes

- `InfoTooltip.tsx` keeps its `absolute z-50` — tooltips live inside
  inline content, are not part of the global stacking battle, and a
  semantic name like `z-tooltip` would be over-engineering. Leaving
  as-is.
- `TokenSelectModal.tsx` and `TraitEditor.tsx` ship their own
  `z-[100]` divs (custom modals built before the shared primitive
  existed). Out of scope for R039 — they don't collide with the
  navs because they're scrim-and-content in one element with a high
  z-index. Flagging as a follow-up for a future batch that migrates
  them onto the shared `Modal` primitive.
- Public `Modal` API is unchanged for existing callers — only the
  optional `dismissOnBackdrop` prop was added (default `true` →
  legacy behavior).
