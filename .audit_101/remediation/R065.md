## 3. TegridyStaking — `_safeMint` + tokenURI override

- `stake()` uses `_safeMint(msg.sender, tokenId)` (L561).
- `extend()` uses `_safeMint` (L610).
- `tokenURI(uint256)` override (L1494) calls `_requireOwned(tokenId)` first,
  then delegates to `ITegridyURIReader(uriReader).tokenURI(tokenId)` when the
  reader is set; otherwise falls back to `super.tokenURI`. `_requireOwned`
  reverts with OZ `ERC721NonexistentToken` for burned/unminted IDs.
- Owner-only `setUriReader(address)` setter (L1509) for renderer rotation.

## 4. TegridyTokenURIReader — existence gate

- `error URIQueryForNonexistentToken()` declared (L41).
- `tokenURI()` (L54) wraps `staking.ownerOf(tokenId)` in try/catch and reverts
  `URIQueryForNonexistentToken` on failure (L56–60). No zero-rendered SVG for
  non-existent positions — matches EIP-721 mandate.

## 5. TegridyDropV2 — royalty receiver rotation

- Self-rolled 48h timelock (TimelockAdmin was never wired into TegridyDropV2 in
  R023 — doc error in R023.md). Functionally equivalent: value-bound
  propose/execute/cancel triad.
- `proposeRoyaltyReceiverChange(address)` (L472), `executeRoyaltyReceiverChange(address)`
  (L483), `cancelRoyaltyReceiverChange()` (L500). Execute requires re-supplying
  the exact `pendingRoyaltyReceiver` (Compound value-binding).
- `ROYALTY_RECEIVER_DELAY = 48 hours` (L191). `royaltyReceiver` + `royaltyBps`
  mirrored alongside the OZ `_setDefaultRoyalty` slot.
- Errors: `NoPendingRoyaltyChange`, `RoyaltyTimelockNotElapsed`, `RoyaltyValueMismatch`.
- Events: `RoyaltyReceiverProposed`, `RoyaltyReceiverChanged`, `RoyaltyReceiverCancelled`.

  contract-wallet stakers (Safes, vaults) now must implement the hook or revert

  indexers will see a clean revert instead of caching ghost metadata.
