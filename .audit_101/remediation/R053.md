## H-2 — Seaport EIP-712 signature + on-chain ownership

New module `_lib/seaport-verify.js`:
- `verifySeaportSignature({ parameters, signature })` — fetches the offerer's
  current Seaport counter via Alchemy `eth_call` (`getCounter(address)` selector
  `0xf07ec373`), assembles the full `OrderComponents` typed-data message that
  the wallet actually signed (counter is part of the EIP-712 hash but not
  transmitted in the listing row), and calls viem's `verifyTypedData` with
  the canonical Seaport 1.5 domain pinned to verifyingContract
  `0x00000000000000ADc04C56Bf30aC9d3c0aAF14dC`.
- `verifyNftOwnership({ parameters })` — `eth_call` of `ownerOf(uint256)`
  (selector `0x6352211e`) against the NFT contract; result must equal
  `parameters.offerer` (lowercased compare). ERC1155 (itemType 3) is skipped
  pending balanceOf wiring; only ERC721 (itemType 2) listings are gated.
- Both helpers return `{ ok, error }` discriminated unions so the caller can
  map errors to specific HTTP codes.
- Skipping policy mirrors `ratelimit.js`: if `ALCHEMY_API_KEY` is unset and
  `NODE_ENV !== "production"`, helpers return `{ ok: true, skipped: true }`
  so dev/test runs don't need network. **In production** missing key
  → `{ ok: false, error: "rpc-unavailable" }` → 503 fail-closed.

`orderbook.js` `action=create` wires both checks **after** the existing
personal_sign auth check and **before** the duplicate-listing dedupe / DB
insert. Errors are mapped:
- `rpc-unavailable` → 503 (retry path)
- `bad-parameters` / `no-offer-item` / `no-token-id` / `token-not-found` → 400
- `signature-mismatch` / `bad-signature` / `not-owner` → 403

The Seaport signature is read from `order.seaportSignature` first, falling
back to `order.signature` (clients today only send one sig — keep both fields
working so the v2 client can split them).

## H-3 — BigInt-end-to-end price arithmetic

`orderbook.js`:
- New const `MAX_PRICE_WEI = 10n ** 24n` (1,000,000 ETH) imported from
  `seaport-verify.js`. Per-item `startAmount` is rejected if `< 0n` or
  `> MAX_PRICE_WEI` before it joins the sum, so a single griefer can't
  bypass the cap by splitting across consideration rows. Sum is then
  re-checked against the same cap.
- Replaced `Number((priceBig * 1e8) / divisor) / 1e8` with new helper
  `priceWeiToEthNumber(priceWeiBig, decimals)`:
  ```js
  const scaled = (priceWei * 100000000n) / divisor;  // BigInt
  return Number(scaled) / 100000000;                 // safe within cap
  ```
  At `MAX_PRICE_WEI = 10**24` and `decimals = 18`, intermediate `scaled`
  is `10**14`, well under `Number.MAX_SAFE_INTEGER` (`2**53 ≈ 9.007e15`).
- Belt-and-suspenders: if `Number.isFinite(priceEth)` is false after the
  helper, refuse the order rather than write `Infinity` / `NaN` to the
  numeric column.
- `BigInt(...)` calls wrapped in try/catch so non-numeric `startAmount`
  strings produce a clean 400 instead of a 500.

OpenSea proxy mirrors the cap: `MAX_OPENSEA_PRICE_WEI = 10n ** 24n`. Any
`price.current.value` above the cap is replaced with `null` recursively
across the response so a poisoned upstream can't fool the frontend's
`BigInt(priceWei) * 10000n / BigInt(1e18)` rendering pipeline.

## H-1 — OpenSea schema validation + URL sanitization + cache contract

`opensea.js`:
- Per-path zod schemas via `getSchemaForPath()`:
  - `listings/collection/<slug>/best` → `{ listings: [{order_hash?, type?, price?, protocol_data?}], next? }`
  - `(collection|collections)/<slug>/stats` → `{ total?, intervals? }`
  - `events/collection/<slug>` → `{ asset_events: [{event_type?, transaction?, nft?, payment?}], next? }`
  - `orders/ethereum/seaport/(offers|listings)` → `{ orders: [...], next? }`
  - All schemas use `.passthrough()` so OpenSea field additions don't break us;
    only the keys we actually consume are required-shaped.
  - `price.current.value` carries `regex(/^[0-9]+$/)` so `BigInt(value)` later
    can't throw on `"1.5e10"` style inputs.
- `safeParse` runs **before** sanitizers (sanitizers null-out fields, which
  would tank a strict re-parse). Schema mismatch → 502 `Upstream returned
  unexpected shape` + log first 5 issues for triage.
- New module `_lib/url-allowlist.js` ports
  `frontend/src/lib/imageSafety.ts:isAllowedUri` to JS:
  - Allowed schemes: `https://`, `ipfs://`, `ar://`, `data:image/(png|jpeg|webp|gif);base64,...`
  - Explicitly disallowed: `javascript:`, `data:text/html`, `data:image/svg+xml`,
    relative URLs, anything else.
  - `URL_KEYS` set covers every URL field OpenSea can return: `image_url`,
    `image`, `image_original_url`, `image_preview_url`, `image_thumbnail_url`,
    `display_image_url`, `display_animation_url`, `animation_url`,
    `external_url`, `permalink`, `opensea_url`, `metadata_url`, `token_uri`.
  - `sanitizeUrlFields(json)` walks the tree (arbitrary depth) and replaces
    any disallowed value with `null` — frontend already shows a placeholder
    for null images, so existing UI code Just Works.
- Cache contract per finding-4:
  - User-bound paths (`criteria_offers`, `offers/build`,
    `(listings|offers)/fulfillment_data`) and any request carrying `maker=`
    / `offerer=` / `taker=` query param → `Cache-Control: private, no-store`.
  - Public collection-stats GETs → `s-maxage=60, stale-while-revalidate=120`.
  - Other public listings/events GETs → `s-maxage=15, stale-while-revalidate=30`
    (kept tight to track market moves).
  - All POSTs → `private, no-store` regardless of path.

## Out-of-scope follow-ups

- ERC1155 ownership: `verifyNftOwnership` currently skips itemType 3 with
  `{ ok: true, skipped: true }`. None of the three whitelisted contracts
  (Nakamigos, GNSS Art, Jungle Bay) are 1155, so deferring is safe today.
  When a 1155 collection is whitelisted, swap to `balanceOf(owner, id) > 0n`.
- Seaport version pinning: domain hardcodes `version: "1.5"`. If the protocol
  upgrades or we point at Seaport 1.6 (`0x0000000000000068...`), the domain
  needs to update in lockstep with the client constant in
  `frontend/src/nakamigos/constants.js`.
- Counter caching: every create path does a fresh `getCounter` RPC. Per-offerer
  in-memory cache with 30s TTL would cut ~50% of the RPC cost for high-volume
  makers. Defer until volume warrants.
