# R048 ‚Äî API key transport hardening

**Agent:** R048
**Date:** 2026-04-26
**Findings addressed:** 078-H1 ‚Äî API keys flowing through URL path / querystring
risk leakage via Vercel logs, observability tracers, and upstream HTML error
pages that echo the request URL.

## Summary

Moved Alchemy + Etherscan + v1 proxy auth from URL embedding to
`Authorization: Bearer` header. Legacy URL shape is no longer constructed,
and the log sanitizer now scrubs key-shaped URL path segments as defence in
depth against future regressions or stale cached error strings.

## Changes

### `frontend/api/alchemy.js`
- `USE_HEADER_AUTH` gate: when `ALCHEMY_API_KEY` is set, `BASE` and `RPC_BASE`
  drop the `/${KEY}` path segment; `authHeaders()` injects
  `Authorization: Bearer ${KEY}` on every fetch (NFT API + JSON-RPC).
- Demo/dev path retains the path segment so local dev without env vars still
  works against Alchemy's public demo endpoint.
- All `console.error` sites already route upstream text through `logSafe`
  (R057) ‚Äî extended below to scrub path-shaped keys.

### `frontend/api/etherscan.js`
- Switched from v1 (`?apikey=...`) to v2 multichain (`/v2/api` +
  `Authorization: Bearer`) when a real key is set. v2 returns the same
  `{ status, message, result }` shape ‚Äî callers unchanged.
- `chainid=1` query param added (v2 requirement).
- Fallback: keeps v1 + `apikey=` querystring only when no key is configured
  (legacy behaviour for unauth dev). Error log site uses `logSafe(err)`.

### `frontend/api/v1/index.js`
- Same `USE_HEADER_AUTH` pattern for the developer-API proxy: Alchemy NFT
  base URL drops the `/${KEY}` segment; `alchemyAuthHeaders()` injects the
  Bearer header.
- OpenSea path already used `x-api-key` header ‚Äî no change.

### `frontend/api/_lib/logSafe.js` (defence in depth)
Added `KEY_SHAPE_PATH_SEG = /^[A-Z0-9_]{20,}$/` and a path-segment scrub
inside `redactUrlsInString`. Splits `u.pathname` on `/`, replaces any segment
matching the key shape with `[REDACTED]`. This guards against:
- A future contributor reverting to URL-embedded auth
- Stale Vercel-log lines from pre-R048 deploys
- Upstream HTML error pages that echo the original request URL
- Third-party stack traces formatting URLs into `.message`

The threshold is 20 chars, all-caps + digits + underscore ‚Äî matches Alchemy /
Etherscan / OpenSea key alphabet without touching short collection slugs
(`nakamigos`, `gnssart`, etc.) or hex addresses (already redacted by
`HEX_40` rule).
