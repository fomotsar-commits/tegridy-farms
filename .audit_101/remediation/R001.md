# R001 — Etherscan API key leaked via VITE_ prefix (P0-CRITICAL)

**Source finding:** agent 073 — `frontend/.env:3` shipped a real Etherscan key with `VITE_` prefix, bundling it into the client JS and defeating the `/api/etherscan` proxy.

**Pattern applied:** server-only env var (Vercel pattern). VITE_* prefix is reserved for truly-public identifiers (WalletConnect projectId, Supabase anon URL/key with RLS).

## Files touched

| File | Change |
|---|---|
| `frontend/.env` | Removed `VITE_ETHERSCAN_API_KEY` line. The same value already lives under server-only `ETHERSCAN_API_KEY`. Reorganized into Client vs. Server sections. Added prominent warning comment block at top. Did NOT modify the secret value. |
| `frontend/.env.example` | Removed the `VITE_ETHERSCAN_API_KEY` placeholder. Added `ETHERSCAN_API_KEY=` placeholder under the server-only section. Added note explaining why VITE_ETHERSCAN_API_KEY must not be re-added. |
| `frontend/api/etherscan.js` | No change required — already reads `process.env.ETHERSCAN_API_KEY` (line 7). |
| `frontend/src/pages/HistoryPage.tsx` | No change required — already routes through `/api/etherscan` proxy (line 211). The remaining text reference is a historical comment explaining the prior fix. |

## Before / after sketch

**Before (`frontend/.env`):**
```
VITE_WALLETCONNECT_PROJECT_ID=...
VITE_0X_API_KEY=...
VITE_ETHERSCAN_API_KEY=<REDACTED>     <-- bundled into client JS
VITE_SUPABASE_URL=...
VITE_SUPABASE_ANON_KEY=...
ALCHEMY_API_KEY=...
OPENSEA_API_KEY=...
ETHERSCAN_API_KEY=<REDACTED>          <-- server-only (correct)
```

**After (`frontend/.env`):**
```
[security warning header explaining VITE_ prefix bundling]

# Client-side (safe to bundle)
VITE_WALLETCONNECT_PROJECT_ID=...
VITE_0X_API_KEY=...
VITE_SUPABASE_URL=...
VITE_SUPABASE_ANON_KEY=...

# Server-only (Vercel serverless functions only — NEVER VITE_)
ALCHEMY_API_KEY=...
OPENSEA_API_KEY=...
ETHERSCAN_API_KEY=...                 <-- single source of truth
```

## Audit of other VITE_ vars in `frontend/.env`

| Var | Verdict |
|---|---|
| `VITE_WALLETCONNECT_PROJECT_ID` | OK — public projectId, designed to be exposed. |
| `VITE_0X_API_KEY` | Placeholder string `your_0x_api_key_here`, not a real secret. (Note for future: if real key is added later, it must be moved to server-only.) |
| `VITE_SUPABASE_URL` | OK — public URL. |
| `VITE_SUPABASE_ANON_KEY` | OK — anon key with RLS enforcement, designed to be public per Supabase docs. |

No other live secrets currently leak via VITE_ prefix.

## Verification

- `grep -r "VITE_ETHERSCAN_API_KEY" frontend/` returns only the historical comment in `HistoryPage.tsx` — no live code references.
- `grep -r "import.meta.env.VITE_ETHERSCAN" frontend/` returns no matches.
- `frontend/api/etherscan.js` reads `process.env.ETHERSCAN_API_KEY` and the value still exists in `frontend/.env` under that name.
