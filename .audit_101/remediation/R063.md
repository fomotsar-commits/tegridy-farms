## MockUniRouter parameterization

`contracts/test/Audit195_SwapFeeRouter.t.sol::MockUniRouter195` now exposes
`setExchangeRate(address from, address to, uint256 rateBps)` and an internal
`_outFor(from, to, in)` helper. Default behavior is unchanged (rate == 0
falls back to 10000 bps = 1:1) so every prior test passes without edits.

All three swap functions now route through `_outFor` and enforce
`out >= amountOutMin` ("INSUFFICIENT_OUTPUT") so slippage paths are
actually triggerable.

### New tests added (2)

- `test_slippage_routerInsufficientOutput_paramRate` — rate = 9000 bps, asks
  for `9.5 ether` out on a `10 ether` input → router reverts with
  `bytes("INSUFFICIENT_OUTPUT")`.
- `test_paramRate_postFeeOutputMatchesConfiguredRate` — rate = 8000 bps,
  protocol fee = 30 bps, asserts user receives exactly
  `(10e18 * 0.8) - 0.3% = 7.976 ether`. This proves the slippage gate is
  no longer bypassed by the 1:1 collapse.
