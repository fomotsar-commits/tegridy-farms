# R058 — Build-script safety hardening (M-087-1, M-087-2, M-087-3)

**Agent:** R058
**Date:** 2026-04-25
**Mandate:** Bulletproofing — destructive-script gating

## Findings remediated

### M-087-1 — `frontend/scripts/migrate-art-imgs.mjs` was a silent bulk rewriter
**Risk:** Walks every `*.tsx`/`*.jsx` under `frontend/src/` and rewrites in
place. No dry-run, no backup, no dirty-tree check — if the regex regresses,
in-progress work is silently destroyed.

**Fix:**
- Default mode is now **DRY-RUN** (preview only). Pass `--write` to persist.
- `--write` runs `git status --porcelain` first and aborts if the working
  tree under `frontend/src/` is dirty, unless `--force` is also passed.
- Added `--help` output documenting all flags.
- Per-file output now lists replacement counts; final summary lists total
  files and total `<img/>` tags rewritten (or "would be rewritten" in
  dry-run).
- Verified end-to-end: `node frontend/scripts/migrate-art-imgs.mjs` runs
  cleanly in dry-run, reports `0 file(s)` (migration was already complete
  in a prior batch).

### M-087-2 — `scripts/render-og-png.mjs` recommended unpinned `npx -p X@Y`
**Risk:** The recommended invocation
`npx --yes -p @resvg/resvg-js@2 node scripts/render-og-png.mjs` bypasses the
lockfile and pulls a fresh tarball with no integrity pin on every run.

**Fix:**
- `@resvg/resvg-js` was already pinned in root `package.json` `devDependencies`
  at `^2.6.2` and managed by `pnpm-lock.yaml`. Confirmed and documented.
- Removed the `npx --yes -p ...` recommendation from the script docstring
  and from the missing-dep error message. Both now point at
  `pnpm install` + `node scripts/render-og-png.mjs`.
- Updated `docs/WAVE_0_RUNBOOK.md` Step 5 to use the lockfile-pinned flow.
- The earlier note explaining *why* the npx path was removed is preserved
  in the script docstring so future readers don't re-introduce it.

### M-087-3 — `scripts/extract-missing-abis.mjs` clobbered committed file
**Risk:** Default invocation overwrote checked-in `abi-supplement.ts` with
no provenance, no diff gate, and no way for CI to detect drift.

**Fix:**
- New default behavior writes to
  `frontend/src/lib/abi-supplement.generated.ts` (added to `.gitignore`).
  The committed `abi-supplement.ts` is **not** touched without an explicit
  flag.
- `--write` flag promotes the regeneration to the committed file (devs
  review the diff before committing).
- `--check` flag (CI use) fails with exit code 1 if the committed file
  differs from the artifact-derived output. The provenance header is
  stripped before comparison so SHA/date noise doesn't trip CI.
- Provenance header now embedded in every regeneration:
  `Generated from contracts/out/ at git <sha[-dirty]> on <YYYY-MM-DD>`.
  SHA is captured via `git rev-parse --short HEAD`; `-dirty` suffix is
  appended if `contracts/` has uncommitted changes, so reviewers can tell
  whether the SHA is exact.
- `--write` and `--check` are mutually exclusive (validated up front).
- Added matching npm scripts to root `package.json`:
  - `pnpm run extract-abis` (default — writes to `.generated.ts`)
  - `pnpm run extract-abis:write` (promote to committed file)
  - `pnpm run extract-abis:check` (CI drift detection)
- Updated `docs/DEVELOPING.md` (two locations) to point at the new flow.
