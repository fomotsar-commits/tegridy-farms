# R061 — Stateful invariant coverage for 5 critical contracts

**Date:** 2026-04-26
**Mandate:** Bulletproofing — only 3 stateful invariants existed (FuzzInvariant.t.sol on TegridyPair). 21/25 contracts had zero invariants. This batch adds invariants for 5 highest-risk surfaces.

## foundry.toml
`[invariant]` profile (runs=1000, depth=100, fail_on_revert=false) and `[fuzz] runs=10000` already present at branch tip.

## Test files in `contracts/test/invariants/`

- `PairInvariants.t.sol` — pre-existing; fixed `invariant_kGrowsByFeesOnly` math (was using wrong adjusted-K formula that conflated both legs as inputs simultaneously). Plus `LPBalanceSumEqTotalSupply`, `minLiquidityLocked`.
- `StakingInvariants.t.sol` — pre-existing; added `invariant_totalStakedEqSumPositions`. Now: `accruedLEUnclaimedPool`, `totalUnsettledMatchesActor`, `totalStakedBounded`, `totalStakedEqSumPositions`.
- `LendingInvariants.t.sol` — NEW. `collateralValueGteDebtValue` (every open loan has its NFT escrowed); `offerETHEarmarked` (active-offer ETH solvency). Handler: createOffer / cancelOffer / acceptOffer. Wired for the 6-arg TegridyLending (TWAP + sequencerFeed).
- `RevenueInvariants.t.sol` — NEW. `voteWeightConservation` (sum epoch.totalETH == totalDistributed); `ETHSolvency` (earmarked-not-claimed + pending withdrawals <= ETH+WETH on hand). Handler: fund-and-distribute / claim / warp.
- `NFTPoolInvariants.t.sol` — NEW. `priceMonotonic` (spot == initial + delta * net buys−sells); `roundtripNoLoss` (held-count + _heldIds[] sync); `protocolFeesAccrueOnly`. Handler: buy / sell with tracked counter.

Each handler is the only `targetContract`, uses `bound()` to keep call surfaces live without trivial-revert noise.
