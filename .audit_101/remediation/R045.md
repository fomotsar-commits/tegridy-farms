# R045 — Aggregator chainId + uniform slippage + storage prefix mismatch

**Findings (audit-101 agent 065):** H1 (aggregator hard-coded `CHAIN_ID = 1`),
M1 (slippage drift across 7 aggregators biases "best quote"), M4 (eviction
prefix mismatch silently drops writes for kebab-prefixed `tegridy-` keys).

## Files changed

- `frontend/src/lib/aggregator.ts` — H1 + M1 fix
- `frontend/src/hooks/useSwapQuote.ts` — pass connected chainId + user slippage
- `frontend/src/lib/storage.ts` — M4 fix (whitelist both casing conventions)
- `frontend/src/lib/storage.test.ts` — kebab-prefix eviction test +
  `EVICTABLE_PREFIXES` shape test
- `frontend/src/lib/aggregator.test.ts` — new file, 12 tests covering
  chain gating + uniform slippage propagation

## H1 — chainId

`getMetaAggregatorQuotes` and all 7 sub-quote functions now take `chainId:
number` as a required parameter (positional after `sender`). The constant was
renamed from `CHAIN_ID` to `SUPPORTED_CHAIN_ID = 1` so the meaning is explicit
("the chain this codebase has wired contracts for") rather than a magic
default.

Before fetching anything, `getMetaAggregatorQuotes` short-circuits with an
empty result if `chainId !== SUPPORTED_CHAIN_ID`. This is the kill-switch:
even if a caller forgets to gate the wallet network, no aggregator HTTP calls
go out for an unsupported chain, so users on an L2 never see mainnet token
addresses come back as a "best route".

Each successful `AggregatorQuote` is tagged with the `chainId` it was fetched
for. The post-fetch loop drops any quote whose tag doesn't match (defense in
depth — should never trip in practice but cheap insurance).

`useSwapQuote.ts` already imports `useChainId` from wagmi and stores
`chainId` locally; that value is now passed straight to `getAggregatorPrice`
(line 222) and added to the effect's dep array.

## M1 — uniform slippage

Added `maxSlippagePct: number` parameter (default `DEFAULT_MAX_SLIPPAGE_PCT
= 0.5`) at the meta-aggregator level. It's clamped to `[0.05, 50]` and forwarded
to every sub-quote fn.

- **SwapAPI** — accepts fraction; was hard-coded to `0.05` (5%, an order of
  magnitude looser than the others). Now forwards `slippage / 100`.
- **Odos** — accepts percent; was hard-coded to `0.5`. Now forwards `slippage`.
- **Li.Fi** — accepts fraction; previously omitted. Now forwards
  `slippage / 100`.
- **CowSwap, Kyber routes, OpenOcean /quote, ParaSwap /prices** — these
  endpoints don't accept slippage at quote time (it's consumed at
  signing/build). Documented inline; tagged `maxSlippagePct: null` on the
  returned quote so consumers can see at a glance which sources are
  price-only. Comparison stays apples-to-apples because slippage tolerance
  only affects post-quote execution, not the quoted `amountOut`.

Each `AggregatorQuote` carries the `maxSlippagePct` it was fetched with so
ranking logic can be audited downstream.

## M4 — storage prefix mismatch

Live call-sites use both `tegridy_` (snake_case, 43+ occurrences) and
`tegridy-` (kebab-case, in theme/onboarding/price alerts/NFT-finance pools).
The eviction whitelist matched only `tegridy_`, so kebab-prefixed entries
filled quota but were never freed — `safeSetItem` silently returned `false`
("settings not saving").

Added an exported `EVICTABLE_PREFIXES = ['tegridy_', 'tegridy-'] as const` and
helper `isEvictable(key)`. `evictOldEntries` now scans both prefixes; nothing
unrelated to Tegridy is ever touched.

This is intentionally additive (no breaking changes to consumers) — every
existing setItem call-site keeps its current key, but eviction now actually
covers the keys it generates.

  blocked by current single-chain deployment).

  files.
