# R029 — TegridyNFTLending hardening (HIGH-038, L-1, L-6)

**Status:** Remediated. Tests written. Build globally broken in `PremiumAccess.sol` (unrelated to this contract) so `forge test` deferred per mandate.

## Findings addressed
1. **HIGH-038 (agent 038):** Constructor unconditionally wrote 3 mainnet collection literals into `whitelistedCollections`. Non-mainnet deploys auto-trusted whatever lived at those slots (counter-deploy rug surface).
2. **L-6 (agent 007):** `isDefaulted` view (line 567 prior) and `claimDefault` (line 486 prior) used independent grace-window logic — the UI could announce "defaulted" while `claimDefault` still rejected.
3. **L-1 (agent 007):** `calculateInterest` ceilDiv rounds positive elapsed time up to 1 wei, making MIN_DURATION (1 day) a paper guard against effectively-free 1-block flash loans.

## Changes

### `contracts/src/TegridyNFTLending.sol`
- **Constructor (lines 247-270):** Removed the three mainnet literal whitelist writes. Comment block documents the rationale + post-deploy migration step. Whitelisting now flows exclusively through `proposeWhitelistCollection` -> 24h timelock -> `executeWhitelistCollection`.
- **`executeWhitelistCollection` (lines 656-687):** Aave V3 / Compound listing pattern at execute time:
  1. `extcodesize(collection) > 0` (rejects EOAs / self-destructed contracts) — `CollectionNotContract`.
  2. `IERC165(collection).supportsInterface(0x80ac58cd)` (binding ERC-721 gate) — `CollectionNotERC721`.
  3. `IERC721TotalSupply(collection).totalSupply()` best-effort responsiveness probe (tolerated absent — base ERC-721 omits).
  - New interface declaration `IERC721TotalSupply` (lines 13-15).
  - New constant `ERC721_INTERFACE_ID = 0x80ac58cd` (line 64).
  - Validation runs at *execute* time (after 24h timelock) so a counter-deploy between propose/execute cannot bypass it.
- **L-6 unified predicate (lines 617-621):** `_isDefaulted(Loan memory l) private view` is single source of truth. `isDefaulted(uint256)` view delegates to it (lines 605-608); `claimDefault` reverts via the same predicate (line 521). Boundary kept strictly `> deadline + GRACE_PERIOD` in both paths.
- **L-1 minimum-interest floor:**
  - `MIN_INTEREST_AMOUNT = 1e14` (0.0001 ETH ~= ~$0.30) — line 62. Aave V3 stable-rate-premium analogue.
  - `repayLoan` reverts with `InterestBelowMinimum` if calculated interest < floor (line 465).
  - `getRepaymentAmount` view applies the same floor so UI quote matches contract enforcement (lines 592-598).
- New errors: `CollectionNotContract`, `CollectionNotERC721`, `CollectionNotResponsive`, `InterestBelowMinimum` (lines 226-231).

### `contracts/test/TegridyNFTLending.t.sol`
- Added `NotAnERC721` mock contract (codeSize > 0 but `supportsInterface` always false).
- Added 3 R029 tests at end of suite:
  - `test_whitelistCollection_RequiresERC165AndTimelock` — covers EOA rejection, non-ERC721 rejection, premature-execute rejection, and the happy path emitting `CollectionWhitelisted`.
  - `test_isDefaulted_UnifiedWithClaimDefault` — at exactly `deadline + grace` both paths agree NOT defaulted; one second past, both agree defaulted; after-claim flips back to false.
  - `test_minInterest_FloorEnforced` — sub-floor accrual reverts `InterestBelowMinimum`; `getRepaymentAmount` returns `principal + MIN_INTEREST_AMOUNT`; paying quoted amount succeeds.
