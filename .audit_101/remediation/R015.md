## Battle-tested patterns applied

* **Olympus / Tokemak treasury operations:** every public-callable swap derives its minOut from an oracle-consulted TWAP, **not** from caller input. Caller params are downgraded to "tighten-only" semantics ‚Äî they can demand more than the TWAP floor but never less. This closes H-1 / H-2 / H-3 in one stroke and is the same pattern Aave V3 / Lido / Yearn use for treasury rotations.
* **Aave V3 oracle pattern + R003 / TegridyLending consistency:** the TWAP is wrapped in a `getLatestObservation` staleness gate (`OracleStale` typed error if last observation > 2h) so a paused/abandoned oracle cannot be silently used to grant a 0-min swap. Mirrors the existing R003 fix on `TegridyLending._positionETHValue`.
* **Tokemak treasury misdeploy defence:** `lpToken` is cross-checked against `router.factory().getPair(toweli, weth)` at construction. If the canonical factory does not report this pair (returns zero) or reports a different address, the constructor reverts `LPMismatch`. Closes M-2 deterministically ‚Äî not by trust, by math.
* **MEV deadline tightening (Curve / Aerodrome harvest discipline):** `MAX_DEADLINE` reduced from 2 min ‚Üí 1 min. Flashbots Protect inclusion target is the *next* block (~12 s), so 1 min still gives generous headroom for private-mempool relay while halving the public-mempool sandwich window.

## Severity / risk assessment of leaving the on-chain contract until redeploy

* `accumulate()` is `onlyOwner` and the owner is a multisig. Sandwich exposure requires the multisig signers to (a) NOT use Flashbots Protect and (b) use a public-mempool relay. Both are explicit operational mistakes ‚Äî defensive, not in-band.
* `executeHarvestLP()` is `onlyOwner` + 30-day timelock. The 30 d delay alone makes any sandwich economically unfavourable for an attacker (they cannot pre-commit a price manipulation 30 days out cheaply).
* **Net:** the on-chain contract is operationally safe under the existing Flashbots-only policy. The R015 redeploy hardens the contract to *also* be safe under operator error / public-mempool fallback.

The redeploy is therefore HIGH priority but not P0-emergency. It can ship in the next Wave-0 batch alongside the pending VoteIncentives / V3Features / FeeHook patches noted in `project_wave0_pending.md`.
