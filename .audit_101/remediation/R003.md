# R003 — TegridyLending ETH-floor sandwich-manipulable spot oracle (P0-HIGH)

**Source finding:** agents 006 / 031 / 032 — `contracts/src/TegridyLending.sol:715` (`_positionETHValue`) read raw `ITegridyPair.getReserves()` for the optional ETH-denominated collateral floor. The existing test `TegridyLending_ETHFloor.t.sol::test_sandwich_sameBlockManipulation_succeeds` literally proved a same-block sandwich produced an inflated valuation that satisfied the floor check, granting the borrower a loan against effectively undercollateralised stake. `TegridyTWAP.sol` already existed in the repo but was never imported by `TegridyLending`.

**Pattern applied:** Aave V3 oracle pattern — TWAP read with explicit staleness gate. `TegridyTWAP.consult(pair, tokenIn, amountIn, period)` returns a time-weighted output over the requested window; defence-in-depth `OracleStale` revert if the most-recent observation is older than 2h. Window = 30 minutes (Aave V3 default).

## Signature changes

```diff

+ constructor(address _treasury, uint256 _protocolFeeBps, address _weth, address _pair, address _twap)
```

`_positionETHValue` keeps the same internal-view signature `function _positionETHValue(uint256 toweliAmount) internal view returns (uint256)`. Callers (`acceptOffer`) are unchanged.

New external view: `twap()` returns the immutable `ITegridyTWAP` address (auto-generated getter on the public immutable).

New error `OracleStale()` selector exposed for callers/integrators.

## Deploy-script impact

There is **no dedicated `DeployTegridyLending.s.sol` in `contracts/script/`** as of this commit (verified — only `DeployNFTLending.s.sol` covers the sibling generic NFT lender). The historical Wave 0 deploys instantiated `TegridyLending` ad hoc (now-stale `DeployV3Features.s.sol` exists only in earlier worktrees, not in `main`).

When the next deploy script is authored (or the next redeploy happens):
1. Pass the canonical `TegridyTWAP` address as the 5th constructor arg. `TegridyTWAP` is a stateless singleton — see `script/DeployTWAP.s.sol`.
2. **Bootstrap the TWAP for the TOWELI/WETH pair before any lender posts a non-zero `minPositionETHValue`.** Bootstrap = `twap.update(pair)` ≥2 times spaced by `TegridyTWAP.MIN_PERIOD` (15 min). Without bootstrap, `_positionETHValue` reverts with `InsufficientObservations`. Lender offers with `minPositionETHValue = 0` are unaffected (zero is the backward-compatible default — short-circuits before the TWAP is queried).
3. **Run a keeper that calls `twap.update(pair)` at least every 2 hours** to keep observations within `TWAP_MAX_STALENESS`. Otherwise non-zero ETH-floor `acceptOffer` reverts with `OracleStale`. The existing TWAP-update keeper (called out in `DeployTWAP.s.sol` next-steps) covers this.

## Caveats / open issues

1. **Bootstrap dependency on R012 (agent 013):** the audit notes that `TegridyTWAP` bootstrap deviation gaps were flagged. R003 amplifies that dependency — without bootstrap, the ETH-floor path is bricked (revert) for any non-zero floor. The mitigation is graceful (revert, not stuck funds) and the lender opt-in is zero-default, so production lenders can simply not opt in until R012 closes. **Flag R003 as blocked-on R012 for any redeploy that will turn on a non-zero floor.**
2. **Keeper liveness:** `OracleStale` rejects acceptOffer if updates lapse > 2h. The TWAP keeper must be monitored. Recommend an alert on `getLatestObservation(pair).timestamp` lag.
3. **Backward compatibility:** the 5th constructor arg is breaking. Any downstream tooling (Etherscan verification scripts, frontend ABI consumers) that hardcoded the 4-arg signature will fail until updated. The frontend ABI is regenerated from compiled artifacts so this should self-resolve at next build, but flagging here for completeness.
4. **`pair` immutable retained:** the previous spot-reserve path read `pair.getReserves()` directly. We keep `pair` immutable because (a) `consult()` takes the pair address as its first argument, and (b) the constructor still reads `token0()/token1()` on it for orientation snapshotting. The `ITegridyPair` interface is unchanged.
5. **Test infrastructure caveat:** the foundry `via_ir = true` config hoists `block.timestamp` reads when chained. All test warps in the ETH-floor suite now use forge-std `skip()`. Future contributors editing these tests should follow the same convention or risk silent test-time-travel bugs.
