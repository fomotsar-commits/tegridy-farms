# R004 — Pause-state hardening on admin treasury paths

**Mandate:** Bulletproofing — full autonomy. Source findings: agents 019 + 044.
**Severity:** P0-HIGH (unauthorized treasury drain via pause+admin sequence).
**Pattern:** OZ Pausable invariant — when paused, only emergency exits run; cancellation /
lapse / retry that releases or moves funds must respect `whenNotPaused`.

## Per-file modifier changes

### `contracts/src/CommunityGrants.sol`

| Function | Old gating | New gating | Why |
|----------|------------|------------|-----|
| `retryExecution(uint256)` | `onlyOwner nonReentrant` | + `whenNotPaused` | Prevents pause+retry-to-attacker-recipient drain. |
| `cancelProposal(uint256)` | `nonReentrant` | + `whenNotPaused` | Cancel decrements `totalApprovedPending`; under pause this widens the `withdrawable` window in `emergencyRecoverETH(whenPaused)`, enabling the documented pause+cancel-all+drain rug. |
| `lapseProposal(uint256)` | `nonReentrant` | + `whenNotPaused` | Same exposure shape as `cancelProposal` — releases approved-pending into the emergency-recovery surface. |

`emergencyRecoverETH` audited: still gated on `onlyOwner whenPaused`, still subtracts
`totalApprovedPending` from withdrawable. With cancel/lapse now blocked under pause,
`totalApprovedPending` cannot be artificially zeroed mid-pause — invariant restored.

### `contracts/src/TegridyNFTPool.sol`

| Function | Old gating | New gating | Why |
|----------|------------|------------|-----|
| `withdrawETH(uint256)` | `onlyOwner nonReentrant` | + `whenPaused` | Must be emergency-exit only — owner pauses (publicly observable) before draining. |
| `withdrawNFTs(uint256[])` | `onlyOwner nonReentrant` | + `whenPaused` | Same — closes pause-then-drain-NFTs-and-ETH-in-same-tx rug. |

Note on the better-still timelock-gated path: `TegridyNFTPool` is a clone template with a
fixed storage layout already deployed. Adding `TimelockAdmin` inheritance and new pending
state would shift storage slots and break existing clones. Deferred to a versioned
successor — flagged inline in the contract NatSpec (`TODO`). The `whenPaused` gate alone
already kills the same-tx rug primitive: any drain now requires a public pause first,
giving counterparties a window to react.

### `contracts/src/RevenueDistributor.sol`

Per remediation brief: **left alone**. `claim` / `claimUpTo` are correctly gated on
`whenNotPaused` AND `_isStakingPaused()` — the double-gate is intentional (M-10 fix
prevents claiming on corrupted checkpoint data). User explicitly asked to defer the
"bypass staking-pause when distributor is unpaused" admin-callable path; not adding it
without a clearer specification of the intended invariant.

## Frontend / scripts impact

UI flows that call `withdrawETH` / `withdrawNFTs` on a healthy (unpaused) pool will
now revert. Expected — the operator-correct path is `removeLiquidity` for normal
egress, `pause()` then `withdraw*` for emergency exit. UI may need a "pool is paused"
guard on the emergency-withdraw buttons (likely already present given `pause()` is
already on the pool ABI).

## Rug primitive coverage (post-fix)

Documented attack: owner pauses → cancels-all-approved → emergencyRecoverETH drains.
- Cancel under pause now reverts (`whenNotPaused`).
- Lapse under pause now reverts (`whenNotPaused`).
- Retry under pause now reverts (`whenNotPaused`).
- `emergencyRecoverETH` still subtracts `totalApprovedPending`, which can no longer be
  zeroed during the pause window.

Documented attack: pool owner pauses swaps + drains ETH+NFTs in same tx.
- Both withdraw paths now require pause (`whenPaused`), so the pause is observable
  on-chain before any drain. Same-tx rug still possible by an already-malicious owner,
  but the public pause event gives indexers / monitoring tooling a hook to flag the
  exit. Future hardening: wrap behind `TimelockAdmin` (deferred — see note above).
