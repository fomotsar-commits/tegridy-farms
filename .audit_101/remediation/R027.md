## Source state at start of remediation

`contracts/src/TegridyRouter.sol` was **already fully patched** by a prior pass before this retry:

1. **M-1:** `RecipientIsRouter` custom error declared (line 36); `if (to == address(this)) revert RecipientIsRouter();` guard added to all 9 swap entry-points (`swapExactTokensForTokens`, `swapExactETHForTokens`, `swapExactTokensForETH`, `swapTokensForExactTokens`, `swapTokensForExactETH`, `swapETHForExactTokens`, plus the three `*SupportingFeeOnTransferTokens` variants). Timelocked `proposeSweepStuckTokens` / `executeSweepStuckTokens` / `cancelSweepStuckTokens` admin trio added with 48h `SWEEP_DELAY` matching factory governance, gated by `factory.feeToSetter()` (no new owner state).
2. **M-2:** `_validatePathNoCycles` extended (lines 471–494) with a token-level visited check (O(n²) over the path, n ≤ 10 — bounded by `PathTooLong`). Catches A→B→C→A and any longer cycle; pair-level check kept for defence-in-depth. The pre-existing two-element edge case (path[0] == path[1]) is also covered by `_sortTokens`’s `IDENTICAL_ADDRESSES` check.
3. **M-3:** `INIT_CODE_HASH` immutable captured at construction from `keccak256(type(TegridyPair).creationCode)` — exactly the expression the factory uses (`TegridyFactory.sol` line 128: `bytes memory bytecode = type(TegridyPair).creationCode;`), so the hash is **bit-identical** to the salt-deployed bytecode. `_pairForDeterministic` derives the pair address via CREATE2 arithmetic with no factory staticcall. `_validateAllPairsActive` is a single pre-flight loop (one `disabledPairs` staticcall per hop, done **before** the first `safeTransferFrom`) — closes the mid-tx race window where a guardian disabled pair_2 after pair_1 had received tokens. Public `pairFor(tokenA, tokenB)` exposed for off-chain tooling.

The router-side remediation was therefore **already complete on disk**; this retry only needed to add the missing test surface.

## Tests added (`contracts/test/Audit195_Router.t.sol`)

Three tests appended at end of `Audit195RouterTest`:

1. **`test_swap_Rejects_RecipientIsRouter`** — exercises all 9 swap entry-points (3 exact-input, 3 exact-output, 3 fee-on-transfer) with `to = address(router)`. Asserts each reverts with `TegridyRouter.RecipientIsRouter.selector`.
2. **`test_swap_RejectsIndirectCycle`** — creates pairs A↔B, B↔C, C↔A (all distinct), then attempts a 3-hop swap A→B→C→A. Asserts `CyclicPath` revert on the exact-input, exact-output, and fee-on-transfer variants. Pre-fix this would have succeeded and burned ~0.9% of input.
3. **`test_swap_DeterministicPairAddress_NoFactoryRace`** — four-stage proof:
   - Pair address is predictable BEFORE `factory.createPair` is called.
   - `factory.createPair` deploys at exactly the predicted address (validates INIT_CODE_HASH bit-equality vs the agent 003 drift concern).
   - `pairFor(A,B) == pairFor(B,A)` (token-order independence).
   - With the pair flipped to `disabledPairs[true]` mid-test (via `vm.store` slot probe so the test is admin-API-tolerant), `swapExactTokensForTokens` reverts with `PairDisabled` and **alice's tokenA balance is unchanged** — proving the race window is closed (no `safeTransferFrom` fired before the active-pair check).

All three test names match the spec verbatim.

## INIT_CODE_HASH drift coordination (per agent 003)

**Status: SAFE.** Verified by direct grep:

- `TegridyFactory.createPair` (line 128) uses: `bytes memory bytecode = type(TegridyPair).creationCode;`
- `TegridyRouter` constructor (line 101) uses: `INIT_CODE_HASH = keccak256(type(TegridyPair).creationCode);`

Both compile units reference the **same** `TegridyPair` import via the same source path. Solc emits identical creation code for both, so `INIT_CODE_HASH` will match deployed pairs exactly. Constructor also enforces `require(INIT_CODE_HASH != bytes32(0))` to fail-loud on a future refactor that breaks the import graph. If `TegridyPair` bytecode ever drifts (e.g. compiler version bump, new constructor arg), the router MUST be redeployed alongside the new factory — comment block at lines 56–58 of `TegridyRouter.sol` makes this explicit.
