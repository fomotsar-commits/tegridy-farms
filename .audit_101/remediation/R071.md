## M-072-05 — `OwnerAdminPanelV2.tsx` Dutch-auction client-side invariants
- `dutchValidationError = useMemo(...)` (line 129) mirrors `configureDutchAuction` Solidity guards (`TegridyDropV2.sol:441-450`):
  1. `startTime > 0`
  2. `duration > 0`
  3. `startPrice > endPrice` (strict)
  4. `startPrice − endPrice >= duration` (decay step ≥ 1 wei/sec — prevents zero-decay revert)
- Inline `<p role="alert">` surfaces the message in red (line 345). `ExecButton.disabled` includes `dutchValidationError !== null` so submit is blocked. Defensive re-check in `onClick` (line 365) catches a typed-then-rage-click race where the memo lags one tick.
- `parseEther` wrapped in try/catch returns "Start/End must be valid ETH amounts."; `BigInt()` wrapped likewise for unix/duration. No NaN reaches the wallet.
