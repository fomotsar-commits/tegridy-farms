## Findings remediated

### 1. `useGaugeList.ts` — stale gauge list / weights across mutations
**Before:** 60 s polling missed `GaugeAdded`/`GaugeRemoved` and cross-epoch
emission flips. Users saw retired gauges or pre-roll weights for up to a
minute.
**After:** 5 `useWatchContractEvent` hooks, each calling a single
memoised `refetchAll` that triggers `refetchGauges()` + `refetchPairs()`.
`EpochAdvanced` is sourced from `VoteIncentives` (the strongest emission
roll signal) via the existing `VOTE_INCENTIVES_ABI` import.

### 2. `useBribes.ts` — stale claimable / pending withdrawals
**Before:** `claimable` and `pendingTokenWithdrawals` only refreshed every
60 s, so concurrent deposits/claims/votes by other users left the panel
showing numbers that contradicted chain state.
**After:** 5 `useWatchContractEvent` hooks (`BribeDeposited`,
`BribeDepositedETH`, `BribeClaimed`, `GaugeVoted`, `EpochAdvanced`) all
firing the same `refetchAll = refetch + refetchWhitelist + refetchToweli`.

### 3. `useRestaking.ts` — RPC-supplied phantom rewards
**Before:** Hook trusted whatever `pendingTotal()` returned. A malicious
or buggy upstream could quote rewards larger than the contract budget,
encouraging the user to sign a claim that would revert — or, worse,
attaching a phantom number to the UI before signing.
**After:** Two on-chain-bounded sanity caps:
- `pendingBonus ≤ (totalBonusFunded − totalBonusDistributed) × 2`
- `pendingBase ≤ restakedAmount × 2`

If either is exceeded the value is suppressed to 0n, `pendingTotal`
collapses to whatever survives, and `rewardSanityBreach` is set. UI can
then prompt "Verify on-chain" instead of showing the attacker's number.
Combined with `lib/wagmi.ts:fallback({ rank: true })` (added this turn),
the lying upstream also gets demoted at the transport layer so honest
nodes win the next round-trip.

### 4. `useToweliPrice.ts` + `usePriceHistory.ts` — cache poisoning surface
**Before:** Both hooks read pre-existing localStorage entries
(`tegridy_api_price`, `tegridy_price_baseline`, `tegridy_price_history`)
with `JSON.parse` directly. A schema change without a version bump, a
stale entry from a different commit, or a tampered `ts` would silently
anchor the chart.
**After:** Shared `VersionedCacheEntry<T>` schema with `version: 2`,
`data`, and `signedAt`. `readVersionedCacheEntry` rejects:
- Failed parse (wrapped via existing `safeJsonParse`)
- `version !== PRICE_CACHE_VERSION` (instant invalidate + remove)
- `signedAt` not finite, in the future (>60s slack), or >24h old

`writeVersionedCache` always stamps the current `version` and `Date.now()`.
Every old `localStorage.setItem(...{ data, ts })` and every direct
`JSON.parse(getItem(...))` was replaced.

## Battle-tested patterns

- **Curve `GaugeController` listener pattern (R021/R054):** event-driven
  refetch on the same handlers the indexer subscribes to.
- **Convex / Hidden Hand bribe panels (R020):** every bribe-side event
  triggers a per-pair claimable refresh — UI can never lag on-chain.
- **Aave RPC sanity bounds:** cap pending rewards at on-chain budget +
  small overshoot factor; degrade gracefully on suspect reads.
- **Compound localStorage versioning:** every cache key carries its own
  schema version; reads invalidate on mismatch.
- **viem `rank: true`:** standard healthy-RPC selection — already used
  by Curve UI, Velodrome, Aerodrome.

## Constraints honored

- No public hook return shape changed except `useRestaking` (added
  `rewardSanityBreach`).
- localStorage cache changes are forward-compatible: old entries are
  silently dropped; new entries roll out on next fetch.
- Event ABIs added are exact transcriptions of `GaugeController.sol:119-125`
  and `VoteIncentives.sol:241-262` — verified line-for-line against source.
- No `.env` / secret touched.
