# R014 — TimelockAdmin value-binding (audit 027 H-01 / H-02)

## Finding
`contracts/src/base/TimelockAdmin.sol` previously locked only the readiness timer
keyed by an opaque `bytes32`, not the proposed VALUE. Any sidecar slot
(`pendingFeeTo`, `pendingSweepTo`, etc.) could be silently rewritten between
`propose` and `execute`. H-02 specifically: `TegridyFactory.acceptFeeToSetter`
wrote `_executeAfter[FEE_TO_CHANGE] = 0` directly, bypassing `_cancel` and
skipping the `ProposalCancelled` event.

## Battle-tested model
Compound Timelock + MakerDAO DSPause: `txHash = keccak256(target, value, sig,
data, eta)` — execute requires the EXACT proposed parameters or it reverts.

## Fix applied (TimelockAdmin.sol)
1. **`_executeAfter` is now `private`.** All inheritor mutations must flow
   through internal helpers; direct slot writes are impossible.
2. **Value-bound helpers added** (Compound model):
   - `_valueKey(namespace, encodedValue)` —
     `keccak256(abi.encode(namespace, abi.encode(value)))`
   - `_proposeWithValue(namespace, encodedValue, delay)`
   - `_executeWithValue(namespace, encodedValue)` — reverts
     `NoPendingProposal` if value mismatched (no silent swap)
   - `_cancelWithValue(namespace, encodedValue)`
3. **`_forceCancel(key)`** — replaces previous direct-write idiom; always
   emits `ProposalCancelled` so cancellations are auditable.
4. **`_proposalReadyAt(key)`** — read-only accessor for inheritors that need
   the timestamp for typed events / view functions.
5. Legacy `_propose / _execute / _cancel` retained for compat; existing
   inheritor flows continue to work.

## H-02 (TegridyFactory.acceptFeeToSetter:195) — fixed
Before: `_executeAfter[FEE_TO_CHANGE] = 0;` (direct write, no event)
After (line 213): `if (_forceCancel(FEE_TO_CHANGE)) { pendingFeeTo = address(0); ... }`
— canonical `ProposalCancelled` emitted alongside typed `FeeToChangeCancelled`.
