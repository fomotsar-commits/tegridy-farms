# R064 — Pagination & bound-tightening (RETRY)

**Date:** 2026-04-26
**Mandate:** Bulletproofing — pagination and tighter bounds across three
contracts. Storage append-only, no signature changes to existing externals.

## Source changes

### `contracts/src/RevenueDistributor.sol` (MEDIUM)
- `MAX_CLAIM_EPOCHS` lowered **500 → 250** (and `MAX_VIEW_EPOCHS` mirror).
  At ~1-week distribution cadence this still covers ~5 years (well past the
  56-day stale window), while halving worst-case `_calculateClaim` gas. Curve
  FeeDistributor uses 50; we keep 250 to favour UX. `claimUpTo()` paginates
  the long tail.

### `contracts/src/TegridyNFTPoolFactory.sol` (MEDIUM + LOW)
- New storage `mapping(address => bool) public isPool` (appended after
  `_poolsByCollection`, slot 9 — verified via `forge inspect`).
- `createPool` now sets `isPool[pool] = true` atomically with
  `_allPools.push`.
- New error `NotAPool(address pool)`.
- `claimPoolFeesBatch` now `nonReentrant` and `revert NotAPool(pool)`s when
  caller passes any address not deployed by this factory. Per-pool failures
  are still swallowed via `try/catch` for legitimate-pool-but-stuck cases.
- New paginated finders `getBestBuyPoolPaginated(collection, startIdx, count,
  numItems)` and `getBestSellPoolPaginated(...)`. Both delegate to new
  internal `_bestBuyIn` / `_bestSellIn` helpers; the legacy unbounded
  `getBestBuyPool` / `getBestSellPool` now also delegate to those internals
  (zero behavior change).

### `contracts/src/TegridyFactory.sol` (LOW)
- New constant `MAX_PAIRS = 10000` and error `PairLimitReached`.
- `createPair` reverts `PairLimitReached` once `allPairs.length >=
  MAX_PAIRS` (early-revert before any work).
- New view `allPairsPaginated(start, count) returns (address[])` that
  returns `allPairs[start .. start+count)` clamped to total length.
