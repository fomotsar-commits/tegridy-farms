# R034 — Account-Switch Race & ParseEther Crash Hardening

**Agent:** R034
**Mandate:** Bulletproofing — battle-tested defaults
**Findings addressed:** H1, H2, H3, H4 (audit agent 048)

---

## Summary

Four HIGH-severity frontend defects in the staking flow:

| ID  | Defect                                                            | Class                |
| --- | ----------------------------------------------------------------- | -------------------- |
| H1  | `useFarmActions` refs not reset on `address` change               | account-switch race  |
| H2  | `useLPFarming` receipt effect fires for new wallet after swap     | account-switch race  |
| H3  | `useNFTBoost` returns `holdsJBAC=false` during transient renders  | boost preview wrong  |
| H4  | `parseEther(stakeAmount)` crashes the page on `1.`/`1e3`/>18 dec  | ErrorBoundary nuke   |

---

## Files changed

### New
- `frontend/src/lib/safeParseEther.ts` — `safeParseEther`/`safeParseEtherPositive` + `validEtherInput` zod schema. Strict regex `^\d+(\.\d{1,18})?$` rejects trailing dot, scientific notation, >18 decimals, signed/non-numeric.
- `frontend/src/lib/safeParseEther.test.ts` — 22 cases covering happy path + every rejection class.

### Modified
- `frontend/src/hooks/useFarmActions.ts`
  - Added `txAddressRef` snapshot of submitting wallet.
  - Added `useEffect([address])` that wipes `pendingStakeRef`/`txAddressRef` on swap.
  - Receipt effect now fires `trackStake` only when `txAddressRef.current === address`.
  - `approve`/`stake` use `safeParseEtherPositive` → no more `parseEther('1.')` throws.
- `frontend/src/hooks/useLPFarming.ts`
  - Added `txAddressRef`/`lastHandledHashRef` + `useEffect([address])` reset.
  - `isSuccess`/`isTxError` toast effects now drop on address mismatch.
  - `approveLP`/`stake`/`withdraw` use `safeParseEtherPositive`; tag submitting address on every action.
- `frontend/src/hooks/useNFTBoost.ts`
  - Replaced `args: [address!]` with `args: [userAddr]` (ZERO_ADDR fallback).
  - `holdsJBAC`/`holdsGoldCard` are now `boolean | null` (tri-state).
  - `query.enabled = !!address && correctChain`; per-contract `chainId: CHAIN_ID` pin keeps reads on L1 even if wallet is on L2.
  - `boostMultiplier` only credits `+0.5x` on confirmed `true`; `null` falls back to baseline.
- `frontend/src/pages/FarmPage.tsx`
  - Reads `address` from `useAccount`.
  - `useEffect([address])` wipes `lastActionRef`, `receiptShownHashRef`, `submittedDataRef`, `submittedAddressRef`.
  - `handleStake` tags `submittedAddressRef = address`.
  - Receipt effect drops confetti/showReceipt/points when wallet swapped.
  - `nftBonus` only credited on `holdsJBAC === true`.
  - `stakeNeedsApproval` uses `safeParseEtherPositive` instead of raw `parseEther`.
  - `points.logAction(actionType, nft.holdsGoldCard === true)` casts tri-state → strict boolean.
- `frontend/src/hooks/useNFTBoost.test.ts`
  - Updated to assert tri-state semantics (`null` for unknown), added wrong-chain + 0-balance + disconnected cases.

## Behavioural deltas

- Disabled wallet, wrong chain, or pending JBAC read → preview shows un-boosted multiplier instead of falsely promising +0.5x.
- Switching wallets after submit but before confirm → no confetti, no analytics event, no receipt for the new wallet.
- Typing `1.`/`1e3`/`1.1234567890123456789` into the stake input → submit silently disabled instead of ErrorBoundary unmounting the page.
- `useFarmActions.stake('not-a-number', ...)` still throws `Invalid amount` — existing test contract preserved.
