# R028 — TegridyFactory: guardian timelock + stealth ERC-777 (audit 028 H-01 / M-01 / M-03)

## Findings (agent 003, HIGH/MEDIUM)
- **H-01:** `setGuardian` (legacy L346-351) had NO timelock. A compromised
  `feeToSetter` could install a hostile guardian and disable every pair via
  `emergencyDisablePair` in a single transaction.
- **M-01:** CREATE2 salt `keccak256(token0, token1)` omits `chainid` +
  `address(this)`. Cross-chain replay risk. **DEFERRED** — adding chainid
  would change `INIT_CODE_HASH` and invalidate every live pair address
  already deployed/computed off-chain.
- **M-03:** Stealth ERC-777 acknowledged as bypassable in
  `test_F06_stealth_erc777_bypasses_detection`. Add more probe surfaces.

## Battle-tested model
- **Guardian rotation:** OZ `TimelockController` / Compound Timelock — admin
  role changes flow through propose/execute with a delay window. Reuses
  R014's value-bound `TimelockAdmin` so the queued candidate cannot be
  silently swapped between propose and execute.
- **ERC-777 detection:** ERC-777 mandatory views — `isOperatorFor`,
  `defaultOperators`, `granularity` — that pure ERC-20 implementations do
  not define. More probe surfaces = more defense.

## Fix applied (`contracts/src/TegridyFactory.sol`)
### H-01 — Guardian rotation timelock
1. **`setGuardian(address)` deprecated** — now `revert("Use proposeGuardian()")`.
2. **`proposeGuardian(address _guardian)`** — value-bound key
   `keccak256(GUARDIAN_CHANGE, _guardian)`, delay
   `GUARDIAN_CHANGE_DELAY = 24 hours`. Sets `pendingGuardian` sidecar for
   off-chain visibility. Caller: `feeToSetter`. Emits `GuardianProposed`.
3. **`executeGuardian(address _guardian)`** — caller must supply the same
   candidate that was queued; mismatched values revert
   `NoPendingProposal(derivedKey)` (R014 value-binding). Rotates `guardian`,
   clears `pendingGuardian`. Emits `GuardianExecuted` + legacy `GuardianSet`.
4. **`cancelPendingGuardian(address _guardian)`** — abort during the 24h
   window. Caller: `feeToSetter`. Emits `GuardianProposalCancelled`.
5. **`pendingGuardianTime(address)`** — view helper for the execute-after
   timestamp.
6. New constant: `GUARDIAN_CHANGE = keccak256("GUARDIAN_CHANGE")` (namespace).

### M-01 — Documented deferral
NatSpec note retained: chainid omission is forward-hardening for multi-chain
deploy. Adding it would change `INIT_CODE_HASH` and invalidate deployed pair
addresses. Mitigation: each chain runs its own factory.

### M-03 — Stealth ERC-777 probes
`_rejectERC777` extended with two new staticcall probes that pure ERC-20
tokens (and OZ extensions Permit/Pausable/Burnable/Capped) do NOT implement:
1. `defaultOperators()` — mandatory ERC-777 view returning `address[]`.
2. `isOperatorFor(address,address)` — mandatory ERC-777 view returning bool.

Plus the existing ERC-165 + `granularity()` + ERC-1820 (now covering all
three hook hashes: `ERC777Token`, `ERC777TokensRecipient`,
`ERC777TokensSender`). A stealth ERC-777 must drop one of these mandatory
views — at which point it is no longer ERC-777 compliant.
