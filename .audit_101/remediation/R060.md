# R060 — Test coverage for API hardening fixes (R048–R053, R057)

**Date:** 2026-04-25
**Mandate:** Bulletproofing — write regression tests for the security fixes
landed by sibling agents R048 through R053 (and R057 logSafe). Tests are
written before all siblings have written their change logs; the targeted
handlers already carry the R048-R053 hardening so spec is verifiable from
source.

## New test files

### `frontend/api/__tests__/alchemy.test.js`  (new — 8 tests)
- **R048 auth header (2):** Asserts the Alchemy proxy puts the API key in
  `Authorization: Bearer …` for v3 NFT and RPC fetches, NEVER in the URL
  path. Mocks `globalThis.fetch` and inspects `mock.calls[0]`.
- **R049 body cap (1):** Mocks an upstream response with `Content-Length:
  10_000_000`; asserts handler returns 502 + `"Upstream response too large"`.
- **R049 eth_getLogs range (2):** 10000-block delta passes; 10001-block
  delta returns 400 `"Block range too large (max 10000)"`.
- **R050 cache contract (2):** `eth_blockNumber` keeps `s-maxage`;
  `eth_getLogs` switches to `private, no-store`.
- **R050 CORS fallback (1):** Disallowed origin gets NO
  `Access-Control-Allow-Credentials: true` header.

### `frontend/api/__tests__/etherscan.test.js`  (new — 3 tests)
- **R048 auth header (1):** `Authorization: Bearer` set, secret NOT in URL
  or `apikey=` querystring.
- **R048 v2 multichain (1):** Verifies switch to `/v2/api?chainid=1` when a
  real key is configured.
- **R049 block-range cap (1):** `endblock - startblock > 10000` → 400.

### `frontend/api/__tests__/v1.test.js`  (new — 3 tests)
- **R048 auth header (1):** Alchemy fetches use Bearer, key absent from URL.
- **R049 rate-limit (1):** Inject denying `checkRateLimit`; assert 429
  short-circuits before any business logic, with `limit: 20`, `windowSec:
  60`, `identifier: "v1"`.
- **R049 body cap (1):** Mock 10 MB Content-Length; handler returns 500.

### `frontend/api/__tests__/orderbook.test.js`  (new — 6 tests)
- **R053 ownership (2):** Recovered signer ≠ claimed offerer → 403 "Signer
  does not match offerer". Signature recovery throws → 400.
- **R053 cancel ownership (1):** Recovered signer ≠ recorded maker → 403
  "Signer is not the order maker".
- **R053 body cap (1):** Body > 10 KB → 413.
- **Query validation (2):** Missing contract → 400; non-allowlisted
  contract → 403 (open-proxy guard).

### `frontend/api/__tests__/opensea.test.js`  (new — 7 tests)
- **R053 schema enforcement (2):** Non-allowlisted contract in POST body
  offer item → 403 "Contract not supported"; allowlisted → 200.
- **Path traversal (2):** URL-encoded segments and literal `..` rejected
  via the decode-then-check guard from R050/M5.
- **Body cap (1):** > 10 KB → 413.
- **Query validation (2):** Non-allowlisted `asset_contract_address` → 403;
  non-numeric `token_ids` → 400.

### `frontend/api/__tests__/supabase-proxy.test.js`  (extended — +1 test)
- **R051 body cap (1):** Asserts handler exports `config.api.bodyParser
  .sizeLimit ≤ 32kb` so 100 KB bodies are rejected at the framework layer.

  Note: the XFF[last] / wallet-keyed `extractIp` and `buildRateLimitKey`
  tests already live in `frontend/api/_lib/__tests__/ratelimit.test.js`
  (written by the R051 sibling). I avoided duplicating them.

### `frontend/api/__tests__/auth-siwe.test.js`  (extended — +1 test)
The R052 sibling already wrote 14 tests covering H-076-1 (expirationTime/
notBefore required, in-past, > 15min future), M-076-1 (URI host binding),
M-076-2 (Origin required + allowlist), L-076-4 (DELETE rate-limit +
jwt-verify-not-decode), and the auth-me rate-limit. I added:
- **R052 CSRF threat-model documentation (1):** A marker test paired with a
  large block comment that documents how the SIWE login endpoint resists
  CSRF: SameSite=Strict cookie, EIP-4361 signature requirement (the
  signature IS the CSRF token), origin-pinned credentialed CORS, mandatory
  Origin header, single-use atomically-claimed nonce, and 10/min POST rate
  limit. The test exists so the documentation can't be silently deleted in
  a future cleanup.
