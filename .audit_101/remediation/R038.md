## Problem (per agent 074)

Five compounding responsive failures on iPhone 14+ and iPad:

1. 14 `100vh` uses (zero `100dvh`) â†’ iOS Safari URL-bar cropped page wrappers + body. Bottom 80-100px of every page hid behind the toolbar when visible.
2. iPad mini portrait (744px) is below Tailwind's `md:` (768px) breakpoint â†’ it got the cramped phone BottomNav + hamburger drawer instead of the desktop nav, despite having ample horizontal space.
3. `.nav-link` height â‰ˆ 30px (padding 6px 14px). Existing 44px override only matched `.btn-*` classes â€” TopNav links + Footer links failed Apple HIG 44pt minimum.
4. Page wrappers (`-mt-14 min-h-screen`) lacked `overflow-x-hidden` â†’ long addresses on Contracts/History pages triggered horizontal scroll on iPhone widths.
5. `pb-20` (80px) under-cleared 64px BottomNav + 34px iOS home indicator (-18px gap). TowelieAssistant at `bottom-20` overlapped the rightmost BottomNav tab. Wrong-network banner at `top-14` slid under the Dynamic Island.

## Fix

### `frontend/src/index.css`

- Body `min-height`: `100vh; 100dvh` cascade (older Safari falls back to vh).
- New cascade overrides for `.min-h-screen` and `.h-screen` re-apply `100dvh` after Tailwind's `100vh` so every existing `min-h-screen` page wrapper inherits the iOS fix without touching ~30 page files.
- Touch-target rule expanded: `.nav-link` and `footer a` get `min-height: 44px !important; display: flex !important; align-items: center;` at `<= 767px`. `!important` overrides Tailwind's `block` utility shipped on Footer links.
- `.safe-area-content-bottom` media-query breakpoint moved from `<= 767px` to `<= 639px` to match the new `sm:` BottomNav cutoff so iPad portrait doesn't get phantom 80px bottom padding.
- TODO comment on `.pb-safe` updated â€” `viewport-fit=cover` confirmed already in `index.html`.

### `frontend/src/components/layout/AppLayout.tsx`

- Wrong-network banner `top-14` â†’ `top: calc(3.5rem + env(safe-area-inset-top, 0px))` so it sits below the Dynamic Island instead of behind it.
- Page wrapper: removed `pb-20` (now redundant with `safe-area-content-bottom` calc), added `overflow-x-hidden` to prevent long-address horizontal scroll.

### `frontend/src/components/layout/BottomNav.tsx`

- `md:hidden` â†’ `sm:hidden` (hides at 640px+ so iPad mini portrait gets the desktop nav).
- Moved `safe-area-bottom` class off the inner `<div>` and onto the `<nav>` itself via inline `paddingBottom: env(safe-area-inset-bottom, 0px)` â€” clears the 34px iOS home indicator from below the tab row.

### `frontend/src/components/layout/TopNav.tsx`

### `frontend/src/components/TowelieAssistant.tsx`

- Outer wrapper repositioned: `bottom-[calc(5rem+env(safe-area-inset-bottom,0px)+0.5rem)] sm:bottom-[calc(env(safe-area-inset-bottom,0px)+1rem)]`. On phones he sits above BottomNav + home-indicator + 0.5rem gap. On `sm:+` (no BottomNav) he tucks back to bottom-4 over the safe-area inset.

### `frontend/src/components/GlitchTransition.tsx`

- Mobile fallback canvas height `100vh` â†’ `100dvh` so the glitch overlay covers the dynamic viewport on iOS instead of leaving an un-glitched strip below the URL bar.
