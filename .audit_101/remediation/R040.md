# R040 — History tx-record validation + receipt staleness gate

**Source findings:** agent 058 — H1 (HistoryPage indexer-JSON validation),
M5 + L1 (receipt overlay shares unconfirmed/reverted txs, badge hardcoded
to "Mainnet").

**Patterns applied:**
- zod schema for any externally-controlled JSON crossing the wire.
- viem/wagmi `useWaitForTransactionReceipt({ hash, confirmations: 2 })`
  for finality before social-share.
- chainId-derived label (no hardcoded chain names in UI surfaces).

## H1 — HistoryPage indexer validation

`HistoryPage.tsx` previously used a typeof-only `isValidTxRecord` guard
followed by `slice()` truncation. A poisoned `/api/etherscan` response
that kept the field types as strings could still slip through and have
its `tx.hash` interpolated into `getTxUrl(...)` — a 64-byte URL
component reflected back into a clickable `<a href>`. The proxy is
trusted today, but defense-in-depth says we don't trust shape, we check
shape.

**Replacement:**
- New `TxRecordSchema` (`z.object({...})`) enforces:
  - `hash`: `^0x[a-fA-F0-9]{64}$`
  - `to`:   `^0x[a-fA-F0-9]{40}$`
  - `value`: `^\d+$` (decimal-string wei)
  - `timeStamp`: `string` (digits) OR `number` (transformed to digit
    string for forward-compat with non-Etherscan indexers).
  - `functionName` / `isError`: bounded strings with safe defaults so
    `data.result` entries that omit them don't get dropped wholesale.
  - `gasUsed` / `gasPrice`: optional decimal-string wei.
- New `parseTxRecords(input: unknown): TxRecord[]` runs `safeParse` per
  record and drops failures, never throwing. Returned values are safe
  to render directly.
- Both the cache-read path and the `/api/etherscan` response path now
  use `parseTxRecords`. Old `isValidTxRecord` + `truncateTxFields`
  helpers deleted — no callers remain.
- `TxRecord` type is now `z.infer<typeof TxRecordSchema>`, so callers
  see `gasUsed?: string` etc. unchanged.

Tx-link click behaviour preserved — the `getTxUrl(chainId, tx.hash)`
sites in `HistoryPage` keep working, now with a guarantee that `tx.hash`
is a real 0x[64hex] string.

## M5 + L1 — TransactionReceipt overlay

`TransactionReceipt.tsx` used to render the receipt as authoritative
the moment `showReceipt(...)` was called, with a hardcoded "Mainnet"
badge and an unconditional Share-to-X handler that tweeted an explorer
link before the tx was even mined.

**Changes:**
1. Wired `useWaitForTransactionReceipt({ hash, confirmations: 2 })`.
   Derives a tri-state `'pending' | 'confirmed' | 'failed'`:
   - No hash present → `confirmed` (legacy / synthetic receipts).
   - Wagmi error or `receipt.status === 'reverted'` → `failed`.
   - `isSuccess` and `receipt.status === 'success'` → `confirmed`.
   - Otherwise → `pending`.
2. Hardcoded "Mainnet" badge replaced by two badges:
   - Chain label from new `getChainLabel(chainId)` helper in
     `lib/explorer.ts` (covers Mainnet, Sepolia, Holesky, Goerli,
     Optimism, Base + Sepolia, Arbitrum + Sepolia, Polygon, BNB,
     Avalanche, falls back to `Chain {id}` / `Unknown Network`).
   - Status badge with colour-coded background + border, pulses while
     pending, `aria-live="polite"`.
3. Share-to-X gating:
   - `failed` → button is disabled, `aria-disabled`, tooltip
     explains the tx reverted.
   - `pending` → click opens a warning modal: "Wait for confirmation
     before sharing — pending txs can revert" with two actions:
     **Wait** (closes modal) and **Share anyway** (escape hatch — we
     don't fully block since users can still want to share early).
   - `confirmed` → share runs immediately (legacy behaviour).
4. The warning modal lives inside the existing card (absolute overlay,
   `role="alertdialog"`, `aria-labelledby` / `aria-describedby`).
