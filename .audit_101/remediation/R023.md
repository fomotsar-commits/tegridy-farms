**Findings remediated**: H-01 (root rotation race), M-02 (CEI in mint), M-03 (pause coverage)

## H-01 — Merkle root rotation race (HIGH)

**Risk**: `setMerkleRoot()` callable any time → owner (or compromised key) could exclude in-flight allowlist claimers, or atomically rotate-mint-rotate to siphon supply.

**Fix in place**:
- Legacy `setMerkleRoot(bytes32)` removed; replaced by `proposeMerkleRoot` / `executeMerkleRoot` / `cancelMerkleRoot` (TegridyDropV2.sol L400–429).
- 24h timelock via `TimelockAdmin._proposeWithValue / _executeWithValue / _cancelWithValue` (Compound model) — the proposed root is hashed into the proposal key, so `executeMerkleRoot(otherRoot)` reverts with `NoPendingProposal(derivedKey)`. Owner cannot silently swap roots between propose and execute.
- Phase guard `_canRotateMerkleRoot()` (L437–440): rotation permitted only while `mintPhase == CLOSED || CANCELLED || paused()`. ALLOWLIST / PUBLIC / DUTCH_AUCTION revert with `RootRotationBlocked`. Re-checked at execute-time so a close→reopen window during the 24h delay cannot bypass.
- New events: `MerkleRootProposed(newRoot, executeAfter)`, `MerkleRootRotated(oldRoot, newRoot)`, `MerkleRootCancelled(newRoot)`. Initialize-time wiring still emits `MerkleRootChanged` for factory path.
- Constants: `MERKLE_ROOT_CHANGE = keccak256("DROP_MERKLE_ROOT_CHANGE_VB")`, `MERKLE_ROOT_DELAY = 24 hours` (L189–190).

**Storage layout** (clone-template safety): `TimelockAdmin` already in inheritance chain (L20). Only one new mapping slot was introduced by TimelockAdmin in R014; `MERKLE_ROOT_CHANGE` and `MERKLE_ROOT_DELAY` are `constant` so consume no storage. Existing v2 clones keep the same slot layout — no migration required.

## M-02 — `_safeMint` CEI violation (MEDIUM)

**Risk**: receiver `onERC721Received` hook fires while `totalSupply` / `mintedPerWallet` / `paidPerWallet` still reflect pre-mint values; cross-contract observers (oracles, mirrors, bots) see inconsistent state.

**Fix in place** (TegridyDropV2.sol L328–344):
```
uint256 startId = totalSupply + 1;
totalSupply += quantity;
mintedPerWallet[msg.sender] += quantity;
paidPerWallet[msg.sender] += totalCost;

for (uint256 i; i < quantity; ++i) {
    _safeMint(msg.sender, startId + i);
}
```
Counters now bump BEFORE the `_safeMint` loop. `nonReentrant` still blocks self-reentry; CEI ordering ensures any observer reached via the hook sees the post-mint snapshot.

## M-03 — `pause()` did not block `withdraw()` / `cancelSale()` (MEDIUM)

**Risk**: emergency pause halted mints but funds-out paths remained open — defeats the kill-switch.

**Fix in place**:
- `withdraw()` (L513): added `whenNotPaused`.
- `cancelSale()` (L550): added `whenNotPaused`.
- NatSpec updated to document symmetric kill-switch semantics; owners with `pause()` retain full control via `unpause` → action.
