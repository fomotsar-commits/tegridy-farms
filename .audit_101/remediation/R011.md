# R011 — Lending UI accuracy hardening

**Agent:** R011 (remediation), responding to agent 049 P0-HIGH cluster.
**Scope:** Frontend lending UI only. No contract changes.
**Date:** 2026-04-25

## Findings addressed

| ID | Severity | Symptom |
|----|----------|---------|
| HIGH-049-1 | P0-HIGH | `position[0]` (TOWELI) labelled "ETH" and fed to `computeLTV` → every Borrow-tab risk number understated by the TOWELI/ETH ratio. A 200%+ LTV could read as 30%. |
| HIGH-049-3 | P0-HIGH | `getRepaymentAmount` had no `refetchInterval`; cached value sent as `value:` reverts `InsufficientPayment` because pro-rata interest accrues each second. |
| HIGH-049-4 | HIGH | NFTLendingSection countdown only re-evaluated on parent render, drifting many seconds at a time vs LendingSection's 1-Hz tick. |
| M-049-7 | MEDIUM | `MyLoansTab` (NFT) iterated `1..n` while contract `loans[]` is `0..n-1` — first loan invisible, final read OOB-reverts. |

## Source-of-truth verification

* `contracts/src/TegridyStaking.sol:451` — `getPosition` returns `(amount, boostBps, lockEnd, lockDuration, autoMaxLock, canWithdraw)`. **Index 0 is TOWELI** (`Position.amount`, declared at line 96). The "ETH" label was a bug.
* `contracts/src/TegridyLending.sol:447` — `loanId = loans.length` at push; `loanCount() = loans.length`; valid id range is `[0, loanCount)`. Confirmed 0-indexed.
* `contracts/src/TegridyNFTLending.sol:406` — same 0-indexed convention (`if (_loanId >= loans.length) revert InvalidLoanId()`).
* `_positionETHValue` is `internal`, so the frontend converts via the existing `useTOWELIPrice()` price feed (TWAP-resilient already, see `useToweliPrice.ts:165`).

## Changes

### New files

* **`frontend/src/hooks/useCountdown.ts`** — shared 1-Hz countdown hook used by both lending surfaces; exposes `secondsRemaining` so callers can format their own strings without re-implementing the tick logic.

### `frontend/src/components/nftfinance/LendingSection.tsx`

1. Added `useCountdown` import (now sourced from the shared hook); removed the local in-file copy.
2. Added typed `parsePosition(raw)` accessor returning `{ toweliAmount, boostBps, lockEnd, lockDuration, autoMaxLock, canWithdraw }` — eliminates raw tuple-index access at call sites.
3. Re-typed `computeLTV(principal, positionValueEth: number)`; consumers must pass an ETH-denominated number, not a TOWELI-string.
4. `BorrowRow` (line ~979): replaced `position[0]` ETH label with proper conversion: `toweliAmount × priceInEth → positionEthValue`. Both TOWELI display and ETH-equivalent display are now correct; LTV consumes the ETH-equivalent.
5. Added `InfoTooltip` on the "Position Value" label so users understand they are seeing a spot-price-derived ETH equivalent.
6. `LoanRow.getRepaymentAmount` (line ~1453): added `refetchInterval: 12_000` + `refetchOnWindowFocus: true`; capture `refetch` and call it from the `repaySuccess` effect to invalidate the cached quote post-confirmation.

### `frontend/src/components/nftfinance/NFTLendingSection.tsx`

1. Added `useCountdown` import; removed the ad-hoc `now/deadline/timeLeft` locals from `LoanCard`.
2. `LoanCard.getRepaymentAmount` (line ~910): added `refetchInterval: 12_000` + `refetchOnWindowFocus: true`; `refetch` is invoked on `repaySuccess`.
3. `LoanCard.deadlineStr` rewritten on top of `countdown.secondsRemaining` so the displayed time updates every second.
4. `MyLoansTab.loanContracts` (line ~819): `args: [BigInt(i + 1)]` → `args: [BigInt(i)]`. Aligned with `useMyLoans` and the on-chain 0-indexed `loans[]`.
5. `MyLoansTab.myLoans.map`: `id: i + 1` → `id: i` so subsequent `repayLoan(id)` / `claimDefault(id)` calls hit the correct on-chain index.
