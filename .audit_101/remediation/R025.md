## Findings addressed

| ID | Severity | Symptom |
|----|----------|---------|
| H-1 | HIGH | `swapETHForNFTs(tokenIds, ...)` priced via `numItems = tokenIds.length` on the bonding curve, but the buyer freely picked which token IDs to receive. Rarity sniping: a buyer could pay floor-price and walk away with the rarest ID in `_heldIds[]`. Same structural flaw Sudoswap V1 had until V2's `swapETHForAnyNFTs` fix. |
| H-2 | HIGH | `swapNFTsForETH` committed `spotPrice -= delta * numItems` at L248 BEFORE the `safeTransferFrom` loop at L252. ERC721 receiver hooks invoked during the loop observed a post-update `spotPrice`, defeating Checks-Effects-Interactions and creating a same-tx state inconsistency a malicious ERC721 (or any callback path) could exploit for atomic price-oracle manipulation. |
| H-3 | HIGH | Owner-only `syncNFTs(tokenIds[])` registered any NFT the pool already owned into `_heldIds[]`. Combined with `withdrawNFTs`, this was a rugpull surface: a user fat-fingered an unsafe `transferFrom` of a pool-collection NFT, owner re-registered it via `syncNFTs`, then `withdrawNFTs`-ed it to themselves. Same surface also let the pool become a black hole for foreign ERC721s. |

## Battle-tested patterns

* **Sudoswap V2 `swapETHForAnyNFTs(numItems)` (H-1):** buyer pays per-`numItems`, pool decides which IDs leave. `tokenIds` parameter retained as a calldata size hint (ABI-stable for existing integrations) but its values are IGNORED for selection. Pool pops `_heldIds[0]` per item, with swap-and-pop dynamics from prior swaps surfacing the next ID deterministically. Rare IDs cycle the same way as floor IDs ŌåÆ no cherry-pick, no rarity arbitrage at the uniform bonding-curve price.
* **Uniswap V2 / Seaport CEI ordering (H-2):** snapshot the post-trade price (`uint256 newSpotPrice = spotPrice - delta * numItems`), run all external transfers under the consistent pre-state, then commit `spotPrice = newSpotPrice` AFTER the transfer loop and BEFORE the payout. Any hook re-entry into a `view` reads the pre-update price; `nonReentrant` + 10k gas stipend on payout already block write re-entry.
* **Aave V3 / Seaport `rescueERC721(token, tokenId, recipient)` (H-3):** explicit, guarded rescue path that requires `token != address(nftCollection)` (closes the rugpull), explicit `recipient` (off-chain validated against the foreign token's `Transfer` event log), `whenPaused` (R004 emergency posture), owner-only, and emits `EmergencyNFTRescued(token, tokenId, recipient, rescuer)` for public attestability. Pool-collection NFTs sent via unsafe `transferFrom` remain stranded ŌĆö the bug-bounty trade-off is "no owner abuse > strand recovery", and stranding is signaled in NatSpec.
