## Findings addressed

### H-1 — `connect-src` missing Ankr (FIX)
`frontend/src/lib/wagmi.ts:14` uses `https://rpc.ankr.com/eth` as wagmi
fallback transport but `rpc.ankr.com` was not in the CSP `connect-src`
allowlist. Under the documented failure mode (publicnode + llamarpc both
rate-limit), wagmi falls back to Ankr — which CSP would then block, hard-
breaking the dApp at exactly the moment a fallback is needed.

Cross-checked all transports in `frontend/src/lib/wagmi.ts`:
- `https://ethereum-rpc.publicnode.com` — covered by `https://*.publicnode.com`
- `https://eth.llamarpc.com`           — covered by `https://*.llamarpc.com`
- `https://rpc.ankr.com/eth`           — **added** `https://rpc.ankr.com`
- `http()` (wagmi default)             — falls through to other RPCs above

No other wagmi config file exists; `grep -rn createConfig|getDefaultConfig
src/` returned only `lib/wagmi.ts`.

### M-1 — Orphan SHA-256 hash removed (FIX)
`script-src` previously pinned two hashes:
- `sha256-fs/Fksxr9J5Rwod3ET+U0AyQJosZ8lzM4DBNs4NuZfM=` — JSON-LD (still present)
- `sha256-HLYQhrrVIImu5zi4jq79GmRWdTZjdunmdgi6j+sZJ/s=` — orphan (removed)

Re-ran `frontend/scripts/csp-hash.mjs` against current `index.html`. The
script reports exactly **1** inline `<script>` tag (the JSON-LD block) with
hash `fs/Fksxr...`. The previous orphan corresponded to the now-external
theme-init script, which is loaded via `<script src="/theme-init.js">` and
doesn't need a hash. Orphan hashes are not security-critical but are dead
permissions, indistinguishable from a stale CSP allowlist entry.

### M-2 — Vercel preview deploys leak (FIX)
Previous `redirects` array only matched the literal hostname
`tegridyfarms-three.vercel.app`. Branch and commit preview URLs
(`tegridyfarms-git-*.vercel.app`, `tegridyfarms-<hash>-*.vercel.app`) were
not redirected, so unfinished UI was reachable to anyone with the URL.

Added two redirects:
1. `(?<host>.*\.vercel\.app)` with `missing: tegridyfarms.vercel.app` →
   matches every `*.vercel.app` host except the canonical Vercel host,
   redirects to `https://tegridyfarms.xyz/$1` (302 — non-permanent so
   future preview-auth flips can override).
2. `tegridyfarms.vercel.app` → `https://tegridyfarms.xyz/$1` (301 —
   permanent canonical-domain redirect, was already implicit in the
   prior single-host redirect).

This is the lighter-weight battle-tested pattern. The harder pattern
(Vercel Authentication / Password Protection) requires a dashboard
toggle and is out of scope for code-only remediation; called out
in the parent audit handoff.

### M-3 — Dead `rpc.flashbots.net` permission removed (FIX)
`grep -rn flashbots frontend/src/` returns zero matches. Removed
`https://rpc.flashbots.net` from `connect-src`. Verified post-edit
that no `frontend/src/` file references `flashbots`, and `vercel.json`
no longer contains the literal.

### L-1 — Permissions-Policy extended (FIX)
Added five additional deny-list directives matching Google's
recommended baseline for sites that don't use Privacy Sandbox APIs:
- `interest-cohort=()`            — FLoC opt-out
- `browsing-topics=()`            — Topics API opt-out
- `attribution-reporting=()`      — Attribution Reporting API opt-out
- `private-state-token-redemption=()` — Trust Tokens redemption opt-out
- `private-state-token-issuance=()`   — Trust Tokens issuance opt-out

Existing directives (`camera`, `microphone`, `geolocation`, `payment`,
`usb`, `magnetometer`, `gyroscope`, `accelerometer`) preserved.
