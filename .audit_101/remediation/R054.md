# R054 — Indexer Subscription Coverage Expansion

**Agent:** R054 (remediation)
**Source findings:** Agent 039 (DEX volume gap), Agent 084 (subscription coverage audit)
**Date:** 2026-04-26
**Files touched:** `indexer/ponder.config.ts`, `indexer/ponder.schema.ts`, `indexer/src/index.ts`

## Findings addressed

Indexer covered 9/24 contracts and 23/263 events. Top-5 critical gaps closed.

### 1. TegridyPair Swap/Mint/Burn (FIX)
DEX volume + TVL was unrecoverable. Subscribed via Ponder `factory()` helper
keyed on TegridyFactory `PairCreated(token0, token1, pair, allPairsLength)`,
so every child pair is auto-tracked without manual address bookkeeping.
New table `pairEvent` discriminates by `type` (`swap` | `mint` | `burn`) and
keeps amount0/amount1 deltas plus In/Out columns for swaps.

### 2. Pausable surface across 13 contracts (FIX)
Subscribed to `Paused`/`Unpaused` on all 13 pausable contracts. Three
(`PremiumAccess`, `POLAccumulator`, `TegridyNFTLending`) use a dedicated
`PausableOnly` filter to scope the log scan; the other 10 fold pause
events into their existing primary subscription. New `pauseState` table
holds the *current* state per contract for fast UI lookup; `pauseEvent`
is the immutable audit trail. Shared `recordPauseState()` helper centralises
the upsert path so handler bodies stay 6 lines.

### 3. GaugeController full surface (FIX)
Entire contract was deferred (config L419 comment in pre-R054). Now indexes
`Voted`, `VoteCommitted`, `VoteRevealed`, `VoteCommitNftOwnerSnapshot` (R021
M-1), `GaugeAddProposed`, `GaugeAdded`, `GaugeRemoveProposed`, `GaugeFrozen`
(R021 H-1 two-phase), `GaugeRemoved`, `EmissionBudgetProposed`,
`EmissionBudgetUpdated`. Gauge lifecycle state mutates `gauge` table;
audit trail lands in `gaugeEvent`. Commit/reveal flow uses `gaugeVoteCommit`
and `gaugeVoteRevealed` (string-encoded arrays for pglite portability).

### 4. VoteIncentives commit/reveal + refunds (FIX)
Per R020 + R021, added `EpochAdvanced`, `VoteCommitted`, `VoteRevealed`,
`OrphanedBribeRefunded`, `UnvotedBribeRefunded` (existing `BribeDeposited`,
`BribeClaimed`, `GaugeVoted` retained). New `voteIncentivesCommit`,
`voteIncentivesEpoch`, `voteIncentivesRefund` tables. Refunds use a `type`
discriminator (`orphaned` | `unvoted`) so frontend renders one CTA card.

### 5. TegridyRestaking admin reconciliation (FIX)
Per agent 084, `PositionRefreshed`, `BoostRevalidated`, `EmergencyForceReturn`
were silent — dashboard couldn't render the admin-action timeline. New
`restakingAdminAction` table with `type` discriminator unifies all three.

## Schema fixes

- `EarlyWithdrawn` handler now reads all 4 args. The 4th (`penalty`) is
  persisted on `stakingAction.penalty` (nullable, additive — non-early-
  withdraw rows stay `null`). Prior handler silently dropped it.
- `Restaked` now stores `positionAmount` on `restakingPosition`. Field is
  nullable to keep migration painless.
- All new inserts call `.onConflictDoNothing()` keyed on `event.log.id`
  (a stable chainId + block + logIndex tuple). Reorg replays become idempotent.

## Per-contract startBlock

Replaced shared `24500000` with per-contract values from
`contracts/broadcast/*/1/run-latest.json`. Each constant is named
(e.g. `TEGRIDY_STAKING_START`) and cited inline with the broadcast slot.
This trims roughly 316k blocks of unnecessary historical scan for
post-DeployV2 contracts. Two legacy contracts (`TegridyFactory`,
`NFTPoolFactory`) keep `24_500_000` until their broadcast files are
verified by ops.

## Verification

- `cd indexer && npx tsc --noEmit` — **zero errors in project source**
  (`ponder.config.ts`, `ponder.schema.ts`, `src/index.ts`, `ponder-env.d.ts`).
  Vendor errors from `node_modules/{drizzle-orm, @electric-sql/pglite, pg,
  ponder}` are pre-existing toolchain noise unrelated to R054 (these
  packages ship `.d.ts` files that fail strict checks in 5.x; runtime is
  unaffected and Ponder ships its own type-stripped runtime entry).
- Coverage delta: 9 → 14 indexed contracts (+ 3 pause-only filters); 23 →
  ~70 indexed events. Remaining 10 unsubscribed contracts (TegridyDropV2,
  TegridyLaunchpadV2, TegridyTWAP, etc.) tracked separately as low-priority
  follow-ups in agent 084's deferred list — none of those surfaces drive
  on-page UI today.

## Risk

Low. All schema additions are nullable / additive; no existing column
constraints tightened. Per-contract `startBlock` reduction only speeds up
historical sync — does not skip events because each constant is the
deploy block of its contract. Factory subscription for TegridyPair is
the only "live" lookup path; if `PairCreated` is malformed Ponder will
log an ABI mismatch at startup rather than silently miss pairs.
