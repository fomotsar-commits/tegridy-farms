# R010 — ABI Drift vs Forge Artifacts (P0)

**Finding:** P0 (agent 093 / `093_AbiDrift.md`)
**Date:** 2026-04-25
**Agent:** R010
**Status:** FIXED

## Summary

The frontend had three competing ABI sources (`contracts.ts`, `abi-supplement.ts`,
`generated.ts`) plus a fourth in the indexer (`ponder.config.ts`) — none aligned with
the forge-artifact ground truth. `TEGRIDY_STAKING_ABI` shipped 30 of 144 on-chain
functions and declared a phantom `totalPenaltiesRedistributed` getter that does not
exist on-chain. Re-extracted full ABIs from `contracts/out/*.json` via the existing
`scripts/extract-missing-abis.mjs` (added `TegridyStaking.sol` to its MISSING list),
removed the phantom function + its dead hook, added `harvest()` to the V2 pair ABI,
restored TimelockAdmin event overloads in the indexer, and marked `frontend/src/generated.ts`
as deprecated dead code.

## Consumers of the removed hook

`useReadTegridyStakingTotalPenaltiesRedistributed`:
- `Grep` scope = whole repo. **Zero consumers** outside `generated.ts` itself. The hook was only declared in `generated.ts` (which has zero imports anywhere in `frontend/src/` — verified). No frontend code, hook, page, component, or test imports it. Removal is non-load-bearing.
- The phantom function `totalPenaltiesRedistributed` itself is also referenced only in two `contracts/test/*.t.sol` files (Solidity tests, not frontend) and in the audit/spec markdown (`V2_ROADMAP.md`, `.audit_101/*.md`) — flagged below, not fixed.

## Decision: `frontend/src/generated.ts`

Confirmed dead code (zero imports in `frontend/src/`, verified via `Grep "from ['\"][^'\"]*generated['\"]"`). Did NOT delete because:
1. `frontend/wagmi.config.ts:49` declares `out: 'src/generated.ts'` — `npm run wagmi:generate` regenerates it.
2. Deleting could surprise CI / future contributors who expect the artifact.

Action taken: prepended a `@deprecated` JSDoc header explaining the file is dead, listing the canonical replacements (`./lib/contracts`, `./lib/abi-supplement`), and instructing not to add new imports. Phantom-function entry + dead hook removed defensively in case the file is ever revived.

## Out of scope (flagged, not fixed)

- **`generated.ts` events surface (5 events / 14 contracts)** — wagmi codegen drops events without a `useWatchContractEvent` hook. Combined with the indexer's parallel inline ABIs, there is no canonical events ABI in the repo. Recommend a follow-up batch to (a) decide if `generated.ts` is kept and standardize on it, or (b) add a `frontend/src/lib/events.ts` extracted from forge artifacts so the indexer can import from the same source.
- **Indexer pre-existing `node_modules`-level type errors** (pglite/drizzle/ponder peer-dep drift) cause `tsc --noEmit` to fail with hundreds of errors, none of which originate from our source files. R010 changes added zero new errors at the source level.
- **`contracts/test/TegridyStaking.t.sol` and `contracts/test/Audit195_StakingCore.t.sol`** still reference `totalPenaltiesRedistributed` — these are Solidity tests, not frontend. If tests are passing, they likely use a redefined harness; flagged for the contracts team but not in R010 scope.
- **6 dead supplement ABIs** (`POL_ACCUMULATOR_ABI`, `TEGRIDY_FEE_HOOK_ABI`, `TEGRIDY_LP_FARMING_ABI`, `TEGRIDY_NFT_POOL_ABI_FULL`, `TEGRIDY_PAIR_ABI`, `TEGRIDY_TOKEN_URI_READER_ABI`) are exported but unimported — kept because the team explicitly wanted forge-extracted ABIs available even if currently unused.
- **`scripts/extract-missing-abis.mjs` was reverted by a linter/automation** mid-edit when I added a comment; the second add re-applied successfully. Recommend a CI hook that runs `forge build && node scripts/extract-missing-abis.mjs` and fails if the diff is non-empty (per agent 093's recommendation).
- **Lint rule for hooks-using-dead-functions**: a static check that ABI `functionName` literals exist in the bound ABI would have caught the phantom `totalPenaltiesRedistributed` at codegen time. wagmi types do enforce this for typed hooks, but only when the consumer actually uses the hook. The hook itself was never consumed, so the type system never had a chance to flag it.
