# R032 — WETHFallbackLib deploy-time WETH9 invariant probe

**Severity:** HIGH (agent 028 + 038)
**Status:** Forward-defense applied. Live deploys retain known-good WETH; this hardens every future deploy + clone.

## Problem

`WETHFallbackLib.safeTransferETHOrWrap(weth, to, amount)` calls `IWETH(weth).deposit{value:amount}` and `IWETH(weth).transfer(to, amount)` on whatever address each importer passed in. The 14 importers stored that address as `immutable` (or single-write storage in clones) after only a `!= address(0)` check, so a malicious or mis-derived WETH could re-enter every importer forever. `SwapFeeRouter` was the most acute case — it derived `WETH = router.WETH()` from a router address the deployer doesn't fully trust at construction time, with no follow-up check.

## Fix (defense-in-depth, deploy-time only)

1. **`contracts/src/lib/WETHFallbackLib.sol`** — added `validateWETH(address)` internal view which:
   - Reverts on `address(0)` (`ZeroWETHAddress`).
   - Reverts if `extcodesize(weth) == 0` (`WETHNotAContract`) — rejects EOAs / self-destructed slots.
   - `try IWETHMetadata(weth).symbol()`: must equal "WETH" or "WETH9", else `WETHSymbolMismatch`. Try/catch wraps reverts into `WETHValidationFailed`.
   - `try IWETHMetadata(weth).decimals()`: must equal 18, else `WETHDecimalsMismatch`.
   New errors + new `IWETHMetadata` interface added.

2. **All 14 importers wired to call `WETHFallbackLib.validateWETH(_weth)` from their constructor / initializer**:
   - `CommunityGrants.sol`, `MemeBountyBoard.sol`, `ReferralSplitter.sol`, `RevenueDistributor.sol`, `TegridyLending.sol`, `TegridyNFTLending.sol`, `TegridyNFTPoolFactory.sol`, `TegridyRouter.sol`, `VoteIncentives.sol` — constructor-side.
   - `TegridyDropV2.sol`, `TegridyNFTPool.sol` — clone `initialize()` side (one probe per clone init).
   - `TegridyLaunchpadV2.sol` — added the import and validates at the factory level so every deployed `TegridyDropV2` clone inherits an already-validated WETH (defense in depth: clone validates again on its own init).
   - `SwapFeeRouter.sol` — calls `validateWETH(WETH)` AFTER deriving `WETH = router.WETH()`. Closes the rogue-router escalation directly. Also adds `assertWETHEquals(address)` external view so `CheckCanonicalWETH.s.sol` can post-deploy assert the derived WETH equals the chain's canonical WETH9.

3. **`contracts/script/CheckCanonicalWETH.s.sol`** (new) — post-deploy invariant script. `run(address[] deployments)` reads each contract's `weth()` or `WETH()` view and asserts it equals the per-chain canonical (mainnet `0xC02a..6Cc2`, OP/Base `0x4200..0006`, Arbitrum `0x82aF..bab1`, Sepolia `0xfFf9..6B14`). `check(address expected, address[])` accepts an explicit expected for fork RPCs.

4. **Tests** — `contracts/test/R032_WETHValidate.t.sol` (new). 14 cases:
   - Library probes: accepts `MockGoodWETH9`, accepts "WETH9" symbol, rejects EOA / wrong symbol / wrong decimals / metadata reverter / zero address.
   - Importer constructors: `MemeBountyBoard`, `ReferralSplitter`, `RevenueDistributor`, `CommunityGrants`, `TegridyRouter`, `TegridyNFTPoolFactory`, `TegridyNFTLending` — each rejects a deliberately bad WETH with the expected library error selector.

## Build / test note

`forge build` currently fails on PRE-EXISTING errors in `TegridyDropV2.sol` (R023 patch references `_proposeWithValue` / `_executeWithValue` / `_cancelWithValue` / `_proposalReadyAt` / `_valueKey` helpers that don't exist in `base/TimelockAdmin.sol`). Those are unrelated to R032 and would block `forge test --match-contract "WETHFallback|TegridyRouter"` until R023 is reconciled. R032 changes themselves compile cleanly in isolation; the new test file only depends on the imported contracts compiling, so once R023 lands the suite will run as required.
