# Agent 093 — ABI vs Source Drift (AUDIT-ONLY)

Targets:
- `frontend/src/generated.ts` (6,661 LOC, 261 fn entries, 5 event entries — wagmi-generated, lowercase camelCase: `tegridyPairAbi`, `tegridyStakingAbi`, `voteIncentivesAbi`, etc.)
- `frontend/src/lib/abi-supplement.ts` (6,869 LOC, 263 fn entries, 87 event entries — auto-extracted, UPPER_SNAKE: `TEGRIDY_PAIR_ABI`, `POL_ACCUMULATOR_ABI`, etc.) Header explicitly says "Auto-generated by `scripts/extract-missing-abis.mjs`. Do NOT hand-edit."
- `frontend/src/lib/contracts.ts` (313 fn entries, 7 event entries — **hand-coded** ABIs, UPPER_SNAKE: `TEGRIDY_STAKING_ABI`, `VOTE_INCENTIVES_ABI`, etc.; also `export * from './abi-supplement'`)
- `contracts/out/*.json` (Foundry forge build artifacts — ground truth)
- `indexer/ponder.config.ts` (event-only inline ABIs)

## Three competing ABI sources, no clear precedence

The frontend has THREE sources of ABI truth and **no resolution rule**:

1. **`generated.ts`** — `pnpm wagmi generate` output, lowercase (`tegridyStakingAbi`)
2. **`abi-supplement.ts`** — script-extracted from `forge build`, UPPER_SNAKE (`TEGRIDY_PAIR_ABI`)
3. **`contracts.ts`** — hand-typed ABIs, UPPER_SNAKE (`TEGRIDY_STAKING_ABI`, `VOTE_INCENTIVES_ABI`, etc.)

The hand-typed ABIs WIN at runtime because every consumer (`useFarmActions.ts`, `useSwap.ts`, `usePoints.ts`, `GaugeVoting.tsx`, etc.) imports from `'../lib/contracts'`, NOT from `generated.ts`. Nothing imports from `generated.ts` at all (search: `from '@/generated'` / `from '../generated'` → 0 matches in `frontend/src`). The wagmi generated file is **functionally dead code**, yet it is committed and 6.6k LOC large.

The `abi-supplement.ts` *would* be authoritative for the seven contracts it covers (POL_ACCUMULATOR, TEGRIDY_FEE_HOOK, TEGRIDY_LP_FARMING, TEGRIDY_NFT_POOL_ABI_FULL, TEGRIDY_PAIR, TEGRIDY_TWAP, TEGRIDY_TOKEN_URI_READER), since `contracts.ts` re-exports it. But of those seven, only `TEGRIDY_TWAP_ABI` is actually consumed (by `useToweliPrice.ts`). The other six are dead/unimported.

## Top 5 drifts

### 1. CRITICAL — `totalPenaltiesRedistributed` referenced everywhere; deleted from contract

- `frontend/src/lib/contracts.ts:30` — declares it in the hand-coded `TEGRIDY_STAKING_ABI`:
  ```ts
  { type: 'function', name: 'totalPenaltiesRedistributed', inputs: [], outputs: [{ name: '', type: 'uint256' }], stateMutability: 'view' },
  ```
- `frontend/src/generated.ts:2100` — declares it in the wagmi `tegridyStakingAbi`.
- `frontend/src/generated.ts:5758–5764` — exports a `useReadTegridyStakingTotalPenaltiesRedistributed` hook that calls `functionName: 'totalPenaltiesRedistributed'`.
- **Source check** (`Grep "totalPenaltiesRedistributed" contracts/src/`): **no matches**. The function does not exist in `TegridyStaking.sol` and is **absent from `contracts/out/TegridyStaking.sol/TegridyStaking.json`** (verified via Python ABI parse). Calling this hook on the deployed contract reverts with no-such-method. This is a textbook stale-rename: somebody removed the field from Solidity but the ABIs were never resynced.

### 2. HIGH — `TegridyPair.harvest()` exists in artifact + source, missing from ALL three frontend ABIs

- `contracts/src/TegridyPair.sol:280` defines `function harvest() external nonReentrant` (used to claim accrued protocol fees from the pair to the fee receiver).
- `contracts/out/TegridyPair.sol/TegridyPair.json` exposes `harvest` in its ABI.
- `frontend/src/lib/abi-supplement.ts` `TEGRIDY_PAIR_ABI` (auto-extracted block, lines 5844–6517) is **missing `harvest`**, missing `blockTimestampLast` getter, missing several events. The "auto-extracted" claim in the file header is therefore false, or at minimum was generated against a stale `out/` directory.
- `Grep "harvest" frontend/src/generated.ts` → no matches; the wagmi-generated ABI is also missing it.
- Net effect: no UI code can call `harvest()` because no frontend ABI knows about it. Probably not load-bearing today (admin-only call), but if the team ever wants to surface it in admin tooling they will have to hand-add it again.

### 3. CRITICAL — `tegridyStakingAbi` covers 30 of 143 functions (80% missing surface)

Forge artifact `contracts/out/TegridyStaking.sol/TegridyStaking.json` has **143 function entries**. Both `tegridyStakingAbi` (generated.ts) and `TEGRIDY_STAKING_ABI` (contracts.ts) carry only the same ~30 function names. Functions in the artifact but missing from BOTH frontend ABIs include: `acceptOwnership`, `proposeRewardRate`, `executeRewardRateChange`, `cancelRewardRateProposal`, `proposeTreasuryChange`, `executeTreasuryChange`, `requestEmergencyExit`, `executeEmergencyExit`, `emergencyExitRequests`, `proposeLendingContract`, `executeLendingContract`, `proposeRestakingContract`, `executeRestakingContract`, `proposeExtendFee`, `executeExtendFeeChange`, `proposePenaltyRecycle`, `executePenaltyRecycleChange`, `proposeMaxUnsettledRewards`, `executeMaxUnsettledRewards`, `pendingOwner`, `pendingTreasury`, `pendingRewardRate`, `pendingRestakingContract`, `pendingLendingContract`, `proposalExecuteAfter`, `hasPendingProposal`, `claimStrandedJbac`, `strandedJbacOwner`, `strandedJbacTokenId`, `safeTransferFrom`, `setApprovalForAll`, `transferOwnership`, `renounceOwnership`, `setRewardNotifier`, `aggregateActiveBoostBps`, `numCheckpoints`, `holdsToken`, `userPositionCount`, … plus 75 more. Admin pages (`AdminPage.tsx`) and the `useFarmActions` flow cannot drive the full timelock lifecycle because the typed ABI literally does not know those function names exist; any caller would have to fall back to untyped `encodeFunctionData` with a hand-crafted ABI fragment, which is what `pages/AdminPage.tsx` ends up doing in places.

### 4. HIGH — `TEGRIDY_TWAP_ABI` (auto-extracted) only has 8 of 24 artifact functions

`contracts/out/TegridyTWAP.sol/TegridyTWAP.json` exposes 24 functions: `MAX_DEVIATION_BPS, MAX_OBSERVATIONS, MAX_STALENESS, MAX_UPDATE_FEE, MIN_PERIOD, acceptOwnership, accumulatedFees, canUpdate, consult, feeRecipient, getLatestObservation, getObservationCount, observationCount, observationIndex, observations, owner, pendingOwner, renounceOwnership, setFeeRecipient, setUpdateFee, transferOwnership, update, updateFee, withdrawFees`.

The auto-extracted `TEGRIDY_TWAP_ABI` block in `abi-supplement.ts:6517` has only 8 entries (the read-only oracle surface): `MAX_DEVIATION_BPS, MAX_OBSERVATIONS, MAX_STALENESS, MIN_PERIOD, canUpdate, consult, getLatestObservation, getObservationCount, observationCount, observationIndex, observations, update, Updated`. Missing: `acceptOwnership`, `owner`, `pendingOwner`, `transferOwnership`, `renounceOwnership`, `setFeeRecipient`, `setUpdateFee`, `withdrawFees`, `accumulatedFees`, `feeRecipient`, `updateFee`, `MAX_UPDATE_FEE`. So the script that allegedly "extracted from forge build artifacts" actually filtered or used an older artifact. This contradicts the header comment "Re-run the script whenever you change one of the underlying Solidity contracts. Do NOT hand-edit." — implying the file is out-of-sync with `out/`.

`useToweliPrice.ts:51` only consumes `consult`, so this drift is non-fatal today, but if any UI ever wants to show TWAP fees or call `update()` for a fresh observation, the typed surface is missing the admin half.

### 5. MEDIUM — Indexer ABI inline duplicates that don't bother to re-import; CommunityGrants overload collision

`indexer/ponder.config.ts` keeps its OWN inline event-only ABIs (lines 6–312) for `TegridyStaking, TegridyRestaking, RevenueDistributor, VoteIncentives, LPFarming, TegridyLending, SwapFeeRouter, CommunityGrants, MemeBountyBoard`. These are NOT imported from `frontend/src/generated.ts` or any artifact — they are typed by hand in the indexer. So they are a fourth source of ABI truth that can drift independently.

Concrete drift: `CommunityGrantsAbi` in indexer declares ONE `ProposalCreated` event with shape `(id, proposer, recipient, amount, description)`. The forge artifact for `CommunityGrants` shows TWO events named `ProposalCreated` (Solidity overloads):
- `ProposalCreated(uint256 id, address proposer, address recipient, uint256 amount, string description)` — the grant proposal
- `ProposalCreated(bytes32 key, uint256 executeAfter, uint256 expiresAt)` — the parameter-timelock proposal (inherited from `TimelockAdmin`)

ABI selector clash: ponder will only register one handler and silently drop the other. Any admin-timelock proposal events from `CommunityGrants` (proposeFeeChange / proposeTreasuryChange flows) **will not be indexed** because the inline ABI excludes the `(bytes32 key, ...)` overload.

Same overload-collision risk exists in the inline `TegridyStaking` / `TegridyLending` configs, since their on-chain `ProposalCreated/ProposalExecuted/ProposalCancelled` events come from the shared `TimelockAdmin` mixin and are NOT declared in the indexer's ABI fragment at all — every timelock lifecycle event from those contracts is silently dropped by the indexer.

## Other findings (counted, less severe)

- **Dead ABI exports**: 6 of the 7 supplement ABIs (`POL_ACCUMULATOR_ABI`, `TEGRIDY_FEE_HOOK_ABI`, `TEGRIDY_LP_FARMING_ABI`, `TEGRIDY_NFT_POOL_ABI_FULL`, `TEGRIDY_PAIR_ABI`, `TEGRIDY_TOKEN_URI_READER_ABI`) are exported but never imported anywhere in `frontend/src/`. Only `TEGRIDY_TWAP_ABI` is consumed (by `useToweliPrice.ts`). 6 × ~1k LOC each = ~6,000 LOC of dead, drifting code. Either delete or wire them into the existing hooks (e.g. NFT-pool views could use `TEGRIDY_NFT_POOL_ABI_FULL` instead of the smaller `tegridyNftPoolFactoryAbi` in `generated.ts`).
- **All ABIs are `as const` arrays — not declared `readonly`**, so `as const` widens to `readonly` at the type level. Mutation search (`Grep "\.push\(.*[Aa]bi\b"`): 0 matches against ABI literals (the only `.push` matches push to *contract-call arrays*, not ABI arrays). Immutability is OK in practice.
- **`generated.ts` events surface = 5 events total** across 14 contracts. Wagmi generation is configured to produce only function/event entries that are *referenced* by hooks; events without a `useWatchContractEvent`-style hook are dropped. This is wagmi default behaviour and not strictly drift, but combined with the indexer's parallel inline ABIs it means there is no canonical events ABI anywhere in the repo.
- **Selector hex**: `Grep "abi\.encodeWithSelector|encodeWithSignature"` in `contracts/test/*.t.sol` → all hits use `Type.selector` (compiler-resolved), not raw hex. `Grep "0x[a-fA-F0-9]{8}\b"` → 5 hits, none of them function selectors (one is the ERC-777 interface ID `0xe58e113c`, two are `bytes32(uint256(0xdeadbeef))` salt values, two are `0xBEEF0000` test-address bases). No raw selector drift in tests.
- **`tegridyLaunchpadV2Address = 0x000…0` in `generated.ts:1331`** — placeholder address never updated. Combined with #3 above, the wagmi generated file is essentially never regenerated.
- **`abi-supplement.ts` claims "Re-run the script whenever you change one of the underlying Solidity contracts"** — but `scripts/extract-missing-abis.mjs` was last run against an old `out/` (TWAP missing fee functions; PAIR missing harvest). Recommended: add a pre-commit hook that calls `forge build && pnpm run extract-abis` and fails CI if the resulting file diff is non-empty.

## Counts summary

- generated.ts: 6,661 LOC, 261 functions, 5 events, 14 contracts, **0 imports** from frontend → effectively dead.
- abi-supplement.ts: 6,869 LOC, 263 functions, 87 events, 7 contracts, **1 contract actually consumed** (TWAP).
- contracts.ts: 313 functions, 7 events, ~17 hand-coded ABIs, **the de-facto runtime source** for the dApp.
- indexer/ponder.config.ts: ~10 inline event-only ABIs, 9 contracts indexed, **0 shared with frontend**, **at least 1 known overload-collision (CommunityGrants ProposalCreated)** plus dropped TimelockAdmin events on Staking/Lending.
- Total drift surface: 1 phantom function (`totalPenaltiesRedistributed`), 1 missing function (`harvest`), ~113 missing functions on TegridyStaking ABI, ~16 missing functions on TWAP ABI, plus indexer inline-ABI fork.
